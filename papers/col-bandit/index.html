핵심 요약

Col-Bandit은 ColBERT류 late-interaction retriever의 rerank 단계를 가속하는 query-time 알고리즘이다. 핵심 관찰은 N개의 candidate document와 T개의 query token이 만드는 implicit N×T MaxSim Matrix H에서, 각 문서의 최종 점수 Si가 해당 행의 row-sum이라는 점이다(Eq. 3-5, p.3-4). 기존 exhaustive scorer는 이 행렬의 모든 cell을 계산하지만, 논문은 top-K 문서 집합을 '식별(identify)'하는 데에는 행렬 전체가 필요하지 않다고 본다(Figure 2, p.4). Col-Bandit은 이를 finite-population Top-K identification 문제로 정식화하고, 부분적으로 공개된(partially revealed) cell로부터 문서별 신뢰구간 [LCBi, UCBi]를 유지한다. 이 구간은 support [a,b]에서 나오는 deterministic hard bound와 empirical Bernstein–Serfling inequality(Bardenet and Maillard, 2015) 기반의 variance-adaptive radius r_i^eff(Eq. 7)를 결합한 hybrid interval이다(p.4). 매 round마다 공유된 random permutation π를 따라 B(=4)개의 query token cell을 active set 전체에 대해 한 번의 SIMD pass로 공개하고, UCB가 K번째로 큰 LCB(threshold τr) 아래로 떨어진 문서를 영구 제거한다(Algorithm 1, p.5). 남은 K+M명의 survivor는 Full-MaxSim과 동일한 fused kernel로 전체 T개 token에 대해 정확히 rescore되므로, 최종 점수는 exhaustive baseline과 bit-identical하다(p.5). relaxation knob αef는 compute–fidelity trade-off를 조절하는 단일 파라미터로, αef=1에서는 δ-PAC guarantee(Theorem 1)를 제공하고 배포 기본값 αef=0.2는 formal certificate 없이 calibrated된 운용점을 제공한다(p.5, p.9). 실험적으로 index 수정이나 retraining 없이 BEIR 5개 corpus와 REAL-MM-RAG 4개 corpus(최대 2.68M documents)에서 모든 corpus Overlap@5 ≥ 0.90을 유지하면서 서버 CPU(AMD EPYC 7763, AVX2)에서 single-thread 평균 7.0×, Apple M1 Max(NEON)에서 12.9× wall-clock speedup을 보고한다(p.2, p.8).

해결하려는 문제

논문 주장

late interaction의 query-time 비용 구조

Multi-vector late-interaction retriever(예: ColBERT)는 query와 document를 각각 토큰 임베딩 집합으로 표현한다. single-vector retrieval이 저비용의 dot product 한 번으로 점수를 계산하는 것과 달리, exact late interaction은 문서마다 토큰 단위 연산(MaxSim)의 grid를 평가해야 한다. 논문은 이 계산이 최신 late-interaction 파이프라인에서 병목(bottleneck)이 되는 경우가 많다고 밝히며, ranking fidelity를 희생하지 않으면서 이 연산 수를 줄이는 방법이 필요하다고 주장한다.

PDF p. 1

MaxSim Matrix H와 row-sum 관점의 재정식화

논문은 N개의 candidate document와 T개의 query token 사이의 MaxSim 값 h(d,t)가 implicit한 N×T 크기의 MaxSim Matrix H를 이룬다고 본다(Hi,t = h(di,t)). 각 문서의 late-interaction score Si는 이 행렬에서 해당 행의 합(row-sum)이다(Eq. 3-5). 즉 exhaustive scoring이란 이 행렬의 N×T개 cell을 모두 채우는 일과 같다.

PDF p. 3, 4

핵심 질문: 행렬을 다 채우지 않고 top-K를 식별할 수 있는가

논문이 다루는 문제는 다음과 같다. 행렬 H를 전부 채우지 않고, query-time에 어떤 MaxSim cell만 선택적으로 계산하면 exhaustive scorer와 동일한 top-K 문서 집합 T*K를 통계적으로 신뢰 가능한 수준에서 식별(identify)할 수 있는가. 여기서 목표는 각 문서의 정확한 점수를 복원하는 것이 아니라 '어느 문서가 상위 K개인가'라는 집합 정체(identity)를 맞히는 것이다. 계산 비용은 공개된 cell의 비율인 coverage γ(Ω)=|Ω|/(N×T)로 측정한다(Eq. 6).

기존 방식의 한계

논문 주장

index-time accelerator는 query를 보기 전에 결정을 확정해야 한다

PLAID, ColBERTv2, MUVERA, LEMUR, WARP와 같은 index-time accelerator들은 quantization, centroid 압축, fixed-dimensional encoding을 통해 document representation을 query가 도착하기 전에 압축·투영한다. 논문은 이들이 query를 보기 전에 index-time에 결정을 확정(commit)해야 하므로 보수적으로(conservatively) pruning할 수밖에 없다고 지적한다. 반대로 말하면, 실제 query가 어떤 token으로 이루어졌는지는 index-time에 활용할 수 없는 신호이다.

PDF p. 3

offline token pruning의 영구적 정보 손실

Offline token pruning 계열(Lassance et al., 2021; Tonellotto and Macdonald, 2021; Clavié et al., 2024; MacAvaney et al., 2025)은 index size N 또는 query 길이 T를 줄이기 위해 저효용(low-utility) 토큰을 index-time에 영구적으로 제거한다. 이 결정 역시 query를 보기 전에 offline으로 고정된다.

PDF p. 1, 3

candidate retrieval 이후에도 남는 지배적 비용

표준적인 late-interaction reranking은 N×T개의 MaxSim 값 h(d,t)를 모두 계산한다. 논문은 이 비용이 candidate retrieval 단계를 지나고 나서도 query-time 비용을 지배(dominate)한다고 본다. 즉 후보 축소만으로는 rerank 단계 자체의 비용 구조가 바뀌지 않는다.

PDF p. 3

선행 기법은 exact-MaxSim top-K 자체를 목표로 하지 않는다

논문은 선행 가속화 기법들이 대체로 document representation을 손실 압축(lossily compress)하거나 distill하는 방식을 사용한다고 정리한다. 이들은 근사된 표현 위에서의 점수를 목표로 하는 반면, Col-Bandit은 원본 exact-MaxSim scorer가 내놓았을 top-K 집합 자체를 목표(reference)로 삼는다는 점에서 문제 설정이 다르다.

핵심 contribution

논문 주장

Formulation: progressive scoring 기반 Top-K identification

late-interaction reranking을 progressive scoring framework를 이용한 finite-population Top-K identification 문제로 정식화(cast)하였다. 핵심은 index-time accelerator가 활용할 수 없는 신호인 'query 자체'를 query time에 사용하여, 어떤 MaxSim cell을 계산할지 적응적으로 결정한다는 점이다.

PDF p. 2

Algorithm: Col-Bandit progressive multi-round elimination

empirical Bernstein–Serfling inequality(Bardenet and Maillard, 2015)를 활용해 문서별로 촘촘한(tight) confidence bound를 제공하는 progressive multi-round elimination algorithm인 Col-Bandit을 도입하였다. 조절 가능한 relaxation parameter αef는 αef=1에서(Eq. 7의 단순화된 radius 하에서) δ-PAC certificate를 제공하며, 배포 기본값 αef=0.2는 §5에서 보고하는 Pareto-dominant한 cost–fidelity profile을 유지하는 calibrated relaxation이다.

PDF p. 2, 5

Drop-in Acceleration: index 수정·retraining 불필요

서버 CPU(AMD EPYC 7763, AVX2)에서 7.0×/4.6×(1/8 thread), Apple M1 Max(NEON, 4개 corpus subset)에서 12.9×/10.3×(1/8 thread, K=5)의 wall-clock speedup을 BEIR와 REAL-MM-RAG(총 9개 corpus, 최대 2.68M documents)에서 시연하였다. 테스트된 모든 corpus에서 Overlap@5 ≥ 0.90을 유지하며(CPU-S 기준으로는 HotpotQA-500K를 제외한 모든 corpus에서 ≥0.96), index 수정이나 retraining이 필요 없다. 또한 latency가 K에 대해 sub-linear하게 증가한다고 보고한다.

제안 방식의 전체 구조와 주요 구성요소

논문 주장

전체 구조: progressive matrix completion

Col-Bandit은 문제를 progressive matrix completion으로 다룬다. 토큰 단위 점수는 필요에 따라(on demand) 공개되는 표의 값이며, 목표는 부분적으로 공개된(partially revealed) MaxSim Matrix로부터 exhaustive scorer의 Top-K identity를 추정하되 계산량을 최소화하고 사용자가 지정한 statistical reliability 수준을 유지하는 것이다(Figure 2). Col-Bandit은 vanilla ColBERT 위에서 그대로 동작하는 purely query-time algorithm으로, 문서별 lower/upper confidence bound(LCB/UCB)를 유지하다가 가장 약한 winner의 lower bound가 가장 강한 loser의 upper bound를 초과하는 순간(positive separation gap) top-K를 확정한다.

PDF p. 1, 4

구성요소 1 — MaxSim Matrix H와 observation model

N×T 크기의 implicit MaxSim Matrix H를 정의하고, 공개된 cell 집합 Ω, 문서별 관측 인덱스 Oi = {t : (i,t) ∈ Ω}와 미관측 인덱스 Ui = [T] \ Oi를 유지한다. cell (i,t) 하나를 공개하면 unit cost가 발생하고 Hi,t를 얻는다. 계산 비용은 coverage γ(Ω) = |Ω|/(N×T)로 측정한다(Eq. 6).

PDF p. 3, 4

구성요소 2 — Decision Bounds (hybrid LCB/UCB)

문서 i에 대해 ni = |Oi|개의 cell이 공개되었을 때, 경험적 평균 μ̂i와 partial-sum estimator Ŝi = T·μ̂i를 유지한다. 미관측 항목의 알려진 global support [a,b](예: cosine similarity의 경우 [-1,1])로부터 deterministic hard bound LBi^hard = Σ(t∈Oi) Hi,t + (T-ni)·a, UBi^hard = Σ(t∈Oi) Hi,t + (T-ni)·b를 얻는다. 여기에 empirical Bernstein–Serfling style variance-adaptive radius r_i^eff = αef · T·σ̂i·sqrt(2·log(cNT/δ)/ni)·sqrt(ρni)(Eq. 7, σ̂i는 공개 항목의 경험적 표준편차, ρni는 ni→T일 때 0으로 수렴하는 finite-population correction)를 결합하여 hybrid interval LCBi = max(LBi^hard, Ŝi - r_i^eff), UCBi = min(UBi^hard, Ŝi + r_i^eff)를 구성한다(Eq. 8). 논문은 correctness가 Hi,t ∈ [a,b]에만 의존하며, support를 넓히면 radius만 느슨해질 뿐 elimination의 안전성은 깨지지 않는다고 밝힌다. 또한 per-document token-norm이나 centroid 기반의 더 촘촘한 per-cell bound는 future work로 남긴다.

PDF p. 4, 12

구성요소 3 — Batched Progressive Elimination (Algorithm 1)

per-cell 루프는 dispatch overhead에 지배되므로(MaxSim cell 하나는 몇 개의 FLOP에 불과) Col-Bandit은 active set Ar 위에서 round 단위로 동작한다. 매 round마다 active set의 모든 생존 문서에 대해 공유된 random permutation π의 다음 B개 query token cell을 한 번의 vectorized SIMD pass로 공개하고 bound를 갱신한 뒤, UCB가 K번째로 큰 LCB(threshold τr) 아래로 떨어진 문서를 영구적으로 제거한다. 논문은 이를 'pay-to-prune'으로 설명한다: 각 round는 B·|Ar|번의 평가 비용을 지불하지만, 제거된 문서는 남은 T - rB개 cell을 통째로 절약하므로 active set이 단조 감소하고 누적 예산 Σr B·|Ar|이 실제로 N·T보다 훨씬 작아진다.

PDF p. 4, 5, 27, 28

구성요소 4 — Relaxation knob αef

compute–fidelity trade-off를 조절하는 단일 continuous knob이다. αef=1은 unshrunk Bernstein–Serfling radius(및 Theorem 1의 δ-PAC guarantee)를 회복하고, αef<1은 radius를 좁혀 더 공격적인 elimination을 가능하게 한다(certificate를 coverage와 교환). 배포 기본값은 αef=0.2이다.

PDF p. 2, 5, 8, 26

구성요소 5 — K-margin rescore (안전 여유 M)

elimination loop는 active set이 K가 아니라 K+M개로 줄어들 때까지만 진행된다(기본 M=5). 종료 시 남은 최대 K+M명의 survivor를 Full-MaxSim과 동일한 fused kernel로 전체 T개 query token에 대해 정확히 재채점(exact rescore)하므로, 최종 survivor 점수는 exhaustive baseline과 bit-identical하다. K번째 경계의 tie는 document index로 깨고 정확한 rescore 단계로 연기한다.

PDF p. 5, 30

구성요소 6 — CB-NK kernel (NUMKONG C extension)

Full-MaxSim과 Col-Bandit이 공유하는 fused C kernel로, x86 AVX2/AVX-512와 ARM NEON을 지원한다. elimination loop에서는 int8-quantised coarse pass를, K+M rescore에서는 정밀 fp32 kernel을 사용하는 cache-separated quantized memory layout을 적용한다. round size B=4는 이 fused C kernel의 SIMD register tile 크기에 맞춘 값이다.

단계별 동작 과정

논문 주장

Step 1 — 초기화

모든 N개 candidate를 active set A1 = [N]에 넣고 관측 집합 Ω를 비워둔다. T개 query token에 대한 uniform random permutation π를 고정된 RNG seed로 한 번 추출하며, 생존하는 모든 문서가 이 permutation을 공유한다. 논문은 이 uniform-without-replacement 구조가 Theorem 1을 적용 가능하게 만드는 요소라고 설명한다.

PDF p. 5, 28

Step 2 — 라운드 반복: batch reveal과 bound 갱신

|Ar| > K+M 이고 (r-1)·B < T 인 동안 루프를 돈다. 각 round에서 π의 다음 B개 token을 active set Ar의 모든 문서에 대해 한 번의 vectorized SIMD pass로 공개(reveal)하여 Ω에 추가하고, Eq. 8에 따라 각 문서의 [LCBi, UCBi]를 갱신한다.

PDF p. 5, 28

Step 3 — elimination threshold 계산

elimination threshold τr을 현재 active set Ar에서 K번째로 큰 LCBi 값으로 설정한다. 이는 '적어도 K개의 문서가 τr 이상의 점수를 가진다고 볼 수 있는' 보수적인 하한선 역할을 한다.

PDF p. 5, 28

Step 4 — 영구 제거

UCBi < τr 인 문서를 active set에서 영구적으로 제거하여 Ar+1 = {i ∈ Ar : UCBi ≥ τr}를 구성하고 round counter r을 증가시킨다. 문서는 UCB가 K번째로 큰 LCB 아래로 떨어졌다는 것이 증명될 때만 제거되므로, 1−δ 확률 사건 위에서는 진짜 top-K 구성원이 탈락하지 않는다. 루프는 |Ar| ≤ K+M이 되거나 T개 token이 모두 공개되면 종료한다.

PDF p. 5, 28

Step 5 — survivor의 정확한 rescore

루프 종료 후 남은 survivor 각각에 대해, Full-MaxSim과 동일한 fused MaxSim kernel로 전체 T개 token에 대한 정확한 점수 Si = Σ(t=1..T) Hi,t를 재계산한다. 이 단계 덕분에 최종 점수는 근사값이 아니라 exhaustive baseline과 동일한 값이 된다.

Step 6 — top-K 반환

재채점된 정확한 점수를 기준으로 survivor 중 상위 K개를 선택하여 추정된 top-K 집합 T̂K를 반환한다. 따라서 오류가 발생할 수 있는 유일한 지점은 elimination 단계에서 진짜 top-K 문서가 잘못 제거되는 경우뿐이며, 이는 Overlap@K로 측정된다.

Algorithm 1의 의사코드 흐름

입력은 query Q(T tokens), candidates D(N docs), 그리고 K, M, B, αef, δ이다. (1) r←1, A1←[N], Ω←∅, [T]의 uniform permutation π를 뽑는다. (2) |Ar| > K+M 이고 (r-1)B < T 인 동안: π의 다음 B개 cell을 i ∈ Ar 전체에 대해 한 SIMD pass로 공개하고 Ω에 추가한 뒤 Eq. 8로 [LCBi, UCBi]를 갱신한다. τr ← Ar 위에서 K번째로 큰 LCBi. Ar+1 ← {i ∈ Ar : UCBi ≥ τr}, r ← r+1. (3) 루프 종료 후 각 survivor를 fused MaxSim kernel로 Si ← Σ(t=1..T) Hi,t 로 rescore한다. (4) T̂K ← arg topK(i ∈ Ar) Si 를 반환한다.

PDF p. 5

confidence radius의 설계 (Eq. 7)

radius r_i^eff = αef · T·σ̂i · sqrt(2·log(cNT/δ)/ni) · sqrt(ρni) 는 두 가지 성질을 갖는다. 첫째, σ̂i(공개된 항목의 경험적 표준편차)에 비례하므로 variance-adaptive하다. 즉 query token들에 대해 MaxSim 값이 안정적인 문서는 적은 관측만으로도 좁은 구간을 갖는다. 둘째, finite-population correction ρni가 ni → T일수록 0으로 수렴하므로, 표준 Bernstein bound보다 구간이 빠르게 축소된다(논문은 이를 sampling without replacement 구조 덕분이라고 설명한다). 상수 c는 Bardenet and Maillard (2015, Theorem 4.3)의 Bernstein–Serfling pre-factor이며, 논문은 이를 c=1로 두고 αef에 흡수시킨다.

PDF p. 4, 12, 26

union bound와 optional stopping 처리

per-cell budget을 δ/(NT)로 설정하고 N개 문서와 T개의 reveal count에 대해 union bound를 취하면(Eq. 7의 log(cNT/δ) 항이 이를 반영), 모든 [LCBi, UCBi]가 동시에 valid해진다. 논문은 이 덕분에 데이터에 의존적인 stopping time에서도 구간이 유효하며 optional-stopping 문제를 우회한다고 설명한다.

PDF p. 5, 26, 27

Theorem 1: δ-PAC corner

uniform-without-replacement reveal 하에서 αef=1이면(Eq. 7의 단순화된 radius 기준, Remark 2 참조) P(T̂K = T*K) ≥ 1−δ 라는 δ-PAC guarantee가 성립한다(Theorem 1, 증명은 Appendix C). 증명의 골자는 empirical Bernstein–Serfling interval이 각 행(문서)에 대해 고정된 reveal count에서 성립하고, union bound를 통해 모든 문서·모든 reveal count에서 동시 성립하도록 만든 뒤, 어떤 문서도 UCB가 K번째로 큰 LCB 아래로 증명적으로 떨어지기 전에는 제거되지 않는다는 elimination rule을 적용하는 것이다.

배포 설정과 certificate의 분리

실제 배포 기본값은 αef=0.2, M=5, δ=0.01, B=4이다. αef<1에서는 radius가 좁아져 elimination이 더 공격적으로 이루어지며, 논문은 이것이 'certificate를 coverage와 교환하는' 것이라고 명시한다. 즉 αef=0.2 설정에는 formal certificate가 없지만, 테스트된 모든 corpus에서 exhaustive top-K와의 overlap이 ≥0.90임을 경험적으로 보인다.

논문의 강점

  • 분석자 해석 정확도 손실의 발생 지점이 구조적으로 한 곳(elimination)으로 국한된다. survivor를 전량 exact rescore하므로 최종 점수는 exhaustive baseline과 bit-identical하며, 따라서 '점수 근사 오차'와 '집합 선택 오차'가 섞이지 않는다. 이는 오류 분석과 knob 튜닝을 단순하게 만든다.
    PDF p. 5
  • 분석자 해석 Overlap@K를 headline metric으로 삼아 'exhaustive scorer를 얼마나 충실히 재현하는가'를 encoder 품질과 분리해 평가한다. 이 덕분에 가속 기법의 효과를 end-task 지표의 잡음과 뒤섞지 않고 측정할 수 있다.
    PDF p. 6
  • 분석자 해석 이론과 배포 설정의 관계를 숨기지 않고 단일 knob αef 위에 나란히 배치했다. αef=1은 δ-PAC certificate를 갖는 corner(실측 Overlap@5 ≥ 99%)이고 αef=0.2는 certificate를 포기한 calibrated 운용점임을 명시하여, 실무자가 어떤 보장을 얻고 무엇을 포기하는지 판단할 수 있게 한다.
    PDF p. 5, 8, 9, 24
  • 논문 주장 index 수정, retraining, representation 압축이 전혀 필요 없는 purely query-time 개입이므로 기존 late-interaction 파이프라인에 drop-in으로 적용 가능하다. 논문은 Matryoshka 차원 축소·Ward token pooling과의 조합 실험으로 이 직교성을 실제로 검증한다.
    PDF p. 2, 8, 25
  • 분석자 해석 coverage 절감이 wall-clock으로 이어지는 경로를 실측으로 뒷받침한다. 모든 비교 대상이 동일한 fused kernel을 공유하고 bandit bookkeeping이 총 런타임의 4% 미만이라는 프로파일링 결과가 제시되어, coverage 절감이 오버헤드에 잠식된다는 흔한 실패 모드를 배제한다.
    PDF p. 6, 8
  • 분석자 해석 평가 범위가 넓다. 텍스트(BEIR 5개)와 멀티모달(REAL-MM-RAG 4개), 세 종류의 encoder(ColBERTv2, Jina-ColBERTv2, Granite Vision Embedding), 두 종류의 CPU ISA(AVX2/NEON), 그리고 8.7K에서 2.68M까지의 corpus 규모를 포괄하며, 특히 GPU 메모리 예산을 초과하는 NQ-2.68M 규모에서도 동작한다.
    PDF p. 5, 7, 8
  • 분석자 해석 핵심 ablation(M, αef)이 설계 선택의 필요성을 직접 보여준다. M=0에서 Overlap@5가 0.93–0.94로 떨어지고 M=5에서 ≥0.98로 회복되는 결과는 K-margin rescore가 장식이 아니라 fidelity의 핵심 구성요소임을 입증한다.
    PDF p. 7, 30

한계

  • 저자 명시 한계 Hardware scope: wall-clock 평가가 두 종류의 CPU class(AVX2 서버, NEON 노트북)로 제한된다. GPU-native fused kernel을 구현하지 않았기 때문에 GPU 비교에는 unfused PyTorch baseline을 사용하였다.
    PDF p. 9
  • 저자 명시 한계 Fidelity ceiling: Col-Bandit은 exhaustive MaxSim top-K를 추정할 뿐이며, 기저(underlying) scorer의 품질을 능가할 수 없고 더 낮은 비용으로 그 품질을 복원(recover)할 뿐이다.
    PDF p. 9
  • 저자 명시 한계 Theory vs. deployment: δ-PAC guarantee(Theorem 1)는 full Bernstein–Serfling radius를 쓰는 δ-PAC corner인 αef=1에서만 성립한다. 배포된 αef=0.2는 calibrated된 simplified radius(pre-factor c=1, lower-order term 생략)를 사용하고 int8-quantised 추정값으로 elimination을 결정하므로 formal certificate가 없으며, 이를 논문이 보고하는 cost–fidelity 운용점과 교환한다.
    PDF p. 9
  • 저자 명시 한계 Quora의 end-task metric(nDCG/Recall/MRR)은 harness의 qrels/doc-id mapping 문제로 사용할 수 없어 논문 전체의 task-metric 평균 계산에서 제외되었다(단, 주요 fidelity metric인 Overlap@K는 이 문제의 영향을 받지 않는다).
    PDF p. 9
  • 분석자 해석 elimination 단계는 int8-quantised coarse pass 위에서 결정되므로, 이론이 가정하는 실수 값 관측과 실제 구현 사이에 간극이 있다. 논문 스스로 Overlap@K가 SIMD backend(int8 dot product의 rounding)에 따라 CPU-S와 CPU-M1 간에 달라질 수 있다고 밝히고 있어(예: HotpotQA Overlap@5가 CPU-S 0.93 vs CPU-M1 0.90), fidelity가 하드웨어에 완전히 불변하지는 않는다.
    PDF p. 8, 28, 29
  • 분석자 해석 이득의 크기가 corpus 규모와 점수 분포에 의존할 가능성이 있다. speedup은 큰 corpus(NQ 8.8×)에서 크고 작은 corpus(Quora 4.7×, ArguAna 6.4×)에서 작으며, K를 키울 때의 latency 증가도 작은 ArguAna에서 가장 크다(4.5×). 즉 후보 수가 적거나 상위권 문서 간 점수 차가 작아 separation gap이 늦게 열리는 경우 elimination의 이점이 줄어들 것으로 예상된다.
    PDF p. 7, 8, 24
  • 분석자 해석 maxsim-cpu 대비 약 11배라는 수치는 알고리즘 기여만의 결과가 아니다. 같은 ISA에서 NUMKONG 기반 Full-MaxSim 자체가 maxsim-cpu보다 약 2.3배 빠르므로, 이 비교에는 고도로 최적화된 kernel 구현의 기여가 함께 포함되어 있다. 다른 구현 위에 Col-Bandit을 얹었을 때의 상대 이득은 이보다 작을 수 있다.
    PDF p. 29, 30
  • 분석자 해석 bound의 tightness가 global support [a,b]에 의존한다. 논문 스스로 per-document token-norm이나 centroid 기반의 더 촘촘한 per-cell bound를 future work로 남긴다고 밝히고 있어, 현재의 hard bound는 개선 여지를 남긴 보수적 선택이다. 정규화되지 않은 embedding처럼 support가 넓어지는 상황에서는 radius가 느슨해져 elimination 효율이 떨어질 것으로 예상된다.
    PDF p. 4, 12
  • 분석자 해석 reveal 순서가 uniform random permutation π로 고정되어 있어, 어떤 query token이 판별력이 높은지는 활용하지 않는다. 이 선택은 Theorem 1의 uniform-without-replacement 가정을 만족시키기 위한 것으로 보이며, 결과적으로 token 선택 측면의 적응성은 남겨두고 문서 측면의 적응성(elimination)만 활용하고 있다.

핵심 용어

Late interaction
query와 document를 각각 토큰 단위 임베딩 집합으로 표현하고, 검색 시점에 토큰 간 상호작용(MaxSim)을 계산해 점수를 내는 방식. single-vector 방식과 달리 문서를 하나의 벡터로 압축하지 않으므로 표현력은 높지만 계산 비용이 크다. PDF p. 1, 3
MaxSim / MaxSim Matrix H
query token t에 대해 문서 d의 모든 document token 중 유사도가 최대인 값 h(d,t). 문서의 late-interaction score Si는 모든 query token에 대한 MaxSim의 합(row-sum)이다. N개 문서 × T개 query token의 h(d,t) 값 전체가 implicit한 N×T 행렬 H를 이룬다(Hi,t = h(di,t), Eq. 3-5). PDF p. 3, 4
Coverage γ(Ω)
MaxSim Matrix에서 실제로 계산(공개)한 cell의 비율. γ(Ω) = |Ω|/(N×T) (Eq. 6). Col-Bandit에서 계산 비용의 대리 지표이며, 논문은 Speedup ≈ 100/Cov% 라는 1차 모형을 제시한다. PDF p. 4, 6
Overlap@K
논문의 headline fidelity metric. Overlap@K = |T*K ∩ T̂K| / K로, exhaustive scorer의 top-K 집합 T*K와 Col-Bandit의 추정 집합 T̂K가 얼마나 겹치는지를 측정한다. 두 집합이 완전히 일치할 때만 1이다(Eq. 9). PDF p. 6
Top-K identification (finite-population)
각 문서의 점수를 정확히 추정하는 것이 아니라, '어느 K개 문서가 상위인가'라는 집합의 정체만 맞히는 문제 설정. 후보 문서 수 N이 유한하고 각 문서의 query token 수 T도 유한하므로 finite-population 통계(표본 추출 without replacement) 도구를 쓸 수 있다. PDF p. 2, 4
LCB / UCB (Lower/Upper Confidence Bound)
부분적으로 공개된 cell로부터 각 문서의 총점 Si에 대해 유지하는 신뢰구간의 하한과 상한. Col-Bandit은 support 기반 hard bound와 Bernstein–Serfling radius를 결합한 hybrid interval을 쓴다(Eq. 8). PDF p. 4
Empirical Bernstein–Serfling inequality
Bardenet and Maillard (2015)의 집중 부등식. 표본의 경험적 분산(variance-adaptive)을 이용하고, 유한 모집단에서 복원 없이 추출할 때의 correction(ρni, ni→T이면 0)을 포함하여 표준 Bernstein bound보다 구간이 빠르게 축소된다. Col-Bandit의 radius r_i^eff(Eq. 7)의 기반이다. PDF p. 4, 12
Separation gap
가장 약한 winner의 LCB가 가장 강한 loser의 UCB를 초과하는 상태. 이 gap이 양수가 되면 남은 cell을 계산하지 않아도 top-K 집합이 결정된다는 것이 Col-Bandit의 핵심 직관이다(Figure 2). PDF p. 4
αef (relaxation knob)
confidence radius를 축소하는 계수(αef ∈ (0,1]). αef=1은 unshrunk Bernstein–Serfling radius를 회복하여 δ-PAC guarantee(Theorem 1)를 주고, αef<1은 radius를 좁혀 더 공격적으로 제거한다. 배포 기본값은 0.2로, certificate를 포기하는 대신 낮은 coverage를 얻는 calibrated 운용점이다. PDF p. 5, 8, 9
δ-PAC guarantee
Probably Approximately Correct 보장. 여기서는 P(T̂K = T*K) ≥ 1−δ, 즉 추정된 top-K가 exhaustive top-K와 정확히 일치할 확률이 1−δ 이상임을 뜻한다(Theorem 1, αef=1에서 성립, 실험의 δ=0.01). PDF p. 5, 26
K-margin rescore (margin M)
elimination을 K가 아니라 K+M개 문서가 남을 때까지만 진행하고, 남은 survivor를 전체 T개 token으로 정확히 재채점하는 안전장치(기본 M=5). 덕분에 최종 점수가 exhaustive baseline과 bit-identical해지고, elimination의 borderline 실수가 흡수된다. PDF p. 5, 30
Doc-Uniform
비교 baseline. 적응성 없이 random cell을 정해진 coverage γ만큼 균일하게 공개하는 non-adaptive 방식. 논문에서는 거의 100% coverage를 요구하여 사실상 redundancy upper bound 역할을 한다. PDF p. 5, 6
Ball-carving
비교 baseline. MUVERA(Dhulipala et al., 2024) Appendix C.3의 query-side compression 기법으로, query token들을 묶어 개수를 줄이는 Stage-2 query-time pruning peer. 논문에서 실질적인 비교 대상으로 삼는 baseline이다. PDF p. 5, 6
CB-NK / NUMKONG kernel
Full-MaxSim과 Col-Bandit이 공유하는 fused C MaxSim kernel(x86 AVX2/AVX-512, ARM NEON 지원). elimination 단계는 int8 quantised coarse pass로, K+M rescore는 fp32 정밀 kernel로 처리하는 cache-separated layout을 쓴다. round size B=4는 이 kernel의 SIMD register tile에 맞춰진 값이다. PDF p. 5, 28, 29
Matryoshka / Ward token pooling
Col-Bandit과 조합 가능한 두 압축 축. Matryoshka는 embedding 차원 축소(d=128→64), Ward pooling은 document token 축 압축(k∈{2,4,8})이다. 논문은 이들이 Col-Bandit의 cell-skipping과 곱셈적으로 결합된다고 주장한다(fewer dimensions × fewer tokens × fewer cells). PDF p. 8, 25
