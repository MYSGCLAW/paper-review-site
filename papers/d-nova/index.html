핵심 요약

D-NOVA는 RAG(Retrieval-Augmented Generation) 파이프라인의 지배적 병목인 dense vector retrieval의 데이터 이동(data movement)을 제거하기 위해, 검색 연산 자체를 3D NAND 플래시 배열 내부로 내리는 in-storage 검색 가속기이다. 핵심 아이디어는 NAND의 multi-level cell(QLC)에 INT4로 인코딩된 임베딩 값과 질의로부터 유도된 wordline 전압을 직접 비교하여, MLC 값을 여러 번 읽어내는 반복 read 없이 단순한 binary sensing만으로 유사도를 평가하는 것이다. 이를 위해 저자들은 NAND string의 직렬 on/off 특성에 맞춘 전용 거리 척도 DTS(Dual-bound Tight Similarity Sensing)를 제안한다. DTS는 Upper-Bound Similarity Sensing(UBS)과 그 complement 버전(Comp UBS)을 결합하여, 기존의 느슨한 Lower-Bound Sensing이 유발하던 false positive를 억제하면서 상한·하한을 동시에 좁힌다. DTS는 cosine/L2와 다른 척도이므로 recall 손실이 발생하는데, 이를 offline에서 한 번만 contrastive(InfoNCE) 학습되는 경량 query-side adapter로 보정하여 임베딩을 DTS-정렬 공간으로 투영한다. 전체 검색은 centroid search → coarse Top-K2 search → fine Top-K1 re-ranking의 3단계 IVF 계층 파이프라인으로 수행되며, 각 단계마다 동시에 센싱하는 wordline 수 m을 다르게 설정(stage-aware m-adaptive sensing)하여 정확도와 처리량을 단계별로 절충한다. SSD 컨트롤러는 quickselect/quicksort 같은 경량 정렬과 메타데이터 관리만 담당하고, 최종적으로 문서 텍스트까지 스토리지 내부에서 fetch하여 결과 문서만 host로 전송한다. 논문은 NQ·HotpotQA에서 CPU baseline 대비 33.3–41.7배 처리량, 58.8–71배 에너지 절감, SOTA in-storage 가속기 REIS 대비 4.3–12.1배 처리량, 1.1–1.26배 에너지 이점을 보고한다.

해결하려는 문제

논문 주장

RAG의 병목은 생성이 아니라 검색이다

LLM의 지식은 사전학습 데이터 시점에 고정되어 최신화가 어렵고, RAG는 외부 지식 베이스를 검색해 이를 보완한다. 그러나 질의 하나마다 수십억 개 규모의 고차원 임베딩을 탐색해야 하므로 검색(retrieval) 단계가 시스템 비용을 지배한다. 논문의 Fig. 1(b)는 데이터베이스 규모가 커질수록 이 경향이 심해짐을 보여준다: HotpotQA(16GB)에서는 retrieval이 end-to-end 지연시간의 20.36%지만, wiki_en/NQ(120GB)에서는 69.46%, 1TB 규모의 synthetic dataset에서는 94.26%까지 증가한다. 즉 대규모 벡터 DB에서는 generation이 아니라 retrieval이 전체 파이프라인 지연시간의 대부분을 차지한다.

PDF p. 1, 2

근본 원인은 메모리 계층 전반의 데이터 이동

dense vector retrieval의 비용은 연산량이 아니라 storage → DRAM → processor로 이어지는 메모리 계층 전반의 대규모 데이터 이동에서 발생한다. 벡터 데이터가 계층을 오르내리는 과정에서 지연시간과 에너지가 함께 소모되며, 대역폭이 성능 상한을 결정한다. 저자들이 GPU baseline을 평가에서 제외한 근거도 동일하다 — 병목이 연산 처리량이 아니라 메모리/I-O 대역폭이라는 선행 연구와 일치한다는 판단이다.

PDF p. 1, 2, 10

기존 in-storage 가속기조차 여전히 외부 프로세서에 의존한다

이미 제안된 in-storage 가속기들도 배열(array) 외부의 host 또는 embedded processor에 연산을 의존하기 때문에, 전체 검색 시간의 약 70%를 이 외부 처리 단계에서 소비한다. 결과적으로 대역폭 한계를 완전히 극복하지 못하고 또 다른 memory bottleneck을 만들어낸다. 논문이 목표로 하는 것은 '스토리지 근처'가 아니라 '스토리지 배열 내부'에서 검색을 끝내는 것이다.

기존 방식의 한계

논문 주장

ANN 인덱스 자체의 한계 (HNSW, IVF)

HNSW는 높은 recall 정확도를 달성하지만 irregular pointer-chasing과 예측 불가능한 접근 패턴 때문에 병렬 하드웨어에 효율적으로 매핑하기 어렵다. IVF는 접근 패턴이 훨씬 구조적이어서 하드웨어 친화적이지만, HNSW와 IVF 모두 대량의 벡터 데이터가 메모리/스토리지 계층을 계속 이동해야 한다는 data movement overhead에 근본적으로 제약된다. D-NOVA가 IVF 계열을 택한 이유도 이 구조적 접근 패턴 때문이다.

PDF p. 1

Near-storage 가속기 (RAGX, SmartSSD)

RAGX, SmartSSD 같은 near-storage 가속기는 연산을 스토리지 가까이 옮기지만, 상당한 아키텍처 수정이 필요하고 동기화 및 최종 re-ranking을 위해 여전히 host CPU/GPU에 의존한다. 제한된 on-device 메모리 용량과 엄격한 전력 제약 때문에 경량 데이터 파이프라인 이상으로 확장하기 어렵다. 또한 파이프라인의 여러 단계에 걸쳐 near-storage processor로 중간 데이터를 전송해야 하므로 data movement는 여전히 지배적 병목으로 남는다.

PDF p. 2, 3

SOTA in-storage 가속기 REIS의 재순위화 비용

REIS는 binary quantization을 이용해 coarse ANN search를 NAND 내부에서 수행한다. 그러나 binary 표현의 정확도가 제한적이어서, flash로부터 SSD 컨트롤러/DRAM으로 고정밀 데이터를 끌어와 INT8 re-ranking을 수행하는 값비싼 단계가 반드시 필요하다. 이 재순위화 단계가 전체 검색 시간의 약 70%를 차지하며, MLC read 지연시간과 데이터 이동에 의해 지배된다. 게다가 정확도를 유지하려면 큰 Nnlist와 큰 재순위화 후보 집합이 필요해, 결과적으로 대량의 중간 데이터가 SSD 컨트롤러로 이동해야 한다. D-NOVA는 바로 이 '고정밀 재순위화를 위해 배열 밖으로 데이터를 꺼내야 한다'는 지점을 공략한다.

PDF p. 2, 3

DRAM-SSD 하이브리드 (SPANN, FusionANNS)

SPANN은 centroid를 host DRAM에, 대규모 inverted list를 SSD에 저장하는 하이브리드 설계이며, FusionANNS는 이를 CPU/GPU/SSD 계층으로 확장한다. 그러나 두 경우 모두 후반부 후보 처리(candidate processing)가 host 측에 남아 있어 storage와 host 메모리 간 중간 데이터 이동이 여전히 필요하다.

PDF p. 3

아날로그 PIM 기법을 NAND에 그대로 옮길 수 없다

ReRAM/PCM crossbar 기반 아날로그 PIM은 여러 셀에서 병렬로 흐르는 전류를 아날로그적으로 누적(analog current accumulation)하는 데 의존한다. 반면 NAND flash는 string이 직렬로 연결/차단되는 discrete state로 동작하기 때문에 이러한 crossbar식 전류 누적이 물리적으로 비현실적이다. 또한 NAND 셀은 1-transistor의 극도로 컴팩트한 구조라 셀에 추가 기능을 넣기 어렵고, 이것이 NAND 기반 ISP(in-storage processing)의 역량을 근본적으로 제한해 왔다. 이 제약이 곧 D-NOVA가 '아날로그 누적' 대신 '디지털 on/off 판정'으로 유사도를 정의하게 된 이유다.

핵심 contribution

논문 주장

1) 진정한 in-storage searching and retrieval

D-NOVA는 NAND의 multi-level cell 내부에서 저장된 cell 값과 query-driven wordline 전압을 비교하여 유사도를 배열 내부에서 직접 평가한다. 기존 주변 회로(peripheral circuit)를 재사용해 local group-wise 비교를 수행하므로 하드웨어 오버헤드가 무시할 수준이다. 동작은 완전히 digital하며 on/off 판정에 기반하므로 아날로그 도메인 처리에서 나타나는 non-ideality를 회피한다.

PDF p. 2

2) DTS — NAND string에 맞춘 정확한 거리 척도

NAND string 내부 검색 연산을 위한 맞춤형 거리 척도인 DTS(Dual-bound Tight Similarity Sensing) 프레임워크를 제안한다. DTS는 검색 시 tight한 dual bound(upper/lower)를 적용해 후보 집합을 좁히며, multi-level cell을 읽기 위한 반복적 read 과정 대신 단순한 binary sensing만으로 세밀한 유사도 평가를 가능하게 한다.

PDF p. 2

3) Custom adapter를 통한 데이터베이스 도메인 변환

임베딩을 DTS-정렬 공간으로 투영하는 adapter를 도입해, 널리 쓰이는 cosine/L2 척도와 DTS 사이의 격차를 해소한다. adapter는 document vector에 대한 contrastive objective로 offline에서 단 한 번만 학습되며, in-storage search 시점에는 거의 zero-overhead로 near-full IVF recall을 회복시킨다.

PDF p. 2

4) 제어 가능한 효율-정확도 구성

효율성과 정확도를 절충할 수 있는 controllable knob을 제공한다. block당 병렬로 센싱되는 wordline 수 m을, IVF 각 단계의 정확도 요구사항과 처리 데이터 볼륨에 따라 동적으로 조정하여 단계별 최적화를 수행한다.

PDF p. 2

5) Cross-layer co-design의 실증

회로(sensing)–아키텍처(파이프라인/매핑)–알고리즘(거리 척도/adapter)에 걸친 cross-layer co-design을 통해 대규모 semantic retrieval을 거의 전적으로 in-storage에서 수행하여 데이터 이동을 효과적으로 제거하면서도 높은 정확도를 유지함을 입증했다. NQ와 HotpotQA에서 CPU baseline 및 SOTA(REIS) 대비 각각 33.3–41.7배/4.3–12.1배 높은 처리량과 58.8–71배/1.1–1.26배 낮은 에너지 소비를 달성했다.

제안 방식의 전체 구조

논문 주장

전체 개요

D-NOVA는 SSD 컨트롤러와 D-NOVA 스토리지 칩으로 구성된 완전 통합형 in-storage 가속기이다. centroid search → coarse Top-K2 search → fine Top-K1 re-ranking으로 이어지는 3단계 IVF 기반 계층적 검색 파이프라인을 NAND 메모리 배열 내부에 직접 임베딩하여 실행한다. 이를 가능하게 하는 것이 NAND string 탐색에 특화된 거리 척도 DTS이며, 여기에 임베딩 벡터를 DTS-친화적 도메인으로 매핑하는 경량 contrastive adapter를 결합해 software 수준에 근접한 recall을 유지한다. SSD 컨트롤러는 경량 scoring/sorting 연산만 담당한다.

PDF p. 1, 4

스토리지 계층 구성

NC개의 독립 채널이 각각 MCP 내 ND개 다이에 연결되고, 다이당 NP개 plane이 H-Tree bus로 연결되며, plane마다 3D NAND block(16K BL, 128 WL, 16 SSL) 배열이 배치된다. 이 계층이 채널/plane 수준의 병렬 in-storage 검색을 위한 연산 기본 단위 역할을 한다. 임베딩 벡터와 원문 document chunk는 모두 QLC에 저장되며, 임베딩은 INT4 값으로 각 cell에 인코딩된다. 즉 벡터와 문서가 같은 디바이스 안에 있으므로, 최종 결과 문서까지 host 개입 없이 내부에서 fetch할 수 있다.

PDF p. 4

SSD 컨트롤러의 역할 축소

SSD 컨트롤러는 채널별 flash controller와 SRAM에 연결된 embedded processor로 구성된다. 역할은 quicksort/quickselect 같은 경량 정렬과, cluster centroid 및 Top-K 후보 리스트에 대한 metadata 관리로 한정된다. 각 질의는 컨트롤러가 모든 활성 채널로 브로드캐스트한다. 기존 in-storage 설계에서 컨트롤러가 담당하던 무거운 고정밀 re-ranking이 배열 내부로 흡수되었다는 점이 구조적 차별점이다.

주요 구성요소와 단계별 동작 과정

논문 주장

구성요소 A: DTS(Dual-Bound Tight Similarity Sensing) 엔진

DTS는 두 종류의 센싱을 결합한다. 첫째, Upper-Bound Similarity Sensing(UBS)은 wordline 전압을 VWL=qi+α로 인가하고, m-cell group 내 모든 cell이 ri<qi+α를 만족할 때만 pass로 판정한다(Eq.1). NAND string은 직렬 연결이므로 하나라도 조건을 어기면 string이 차단되고, 이것이 자연스럽게 'all-pass' 논리를 만든다. 둘째, Complement UBS는 4비트 complement 값(ri'=15-ri, qi'=15-qi)에 동일한 all-pass 조건을 적용한다(Eq.3). 기존 Lower-Bound Sensing(LBS, Eq.2)은 group 내 단 하나의 cell만 조건을 만족해도 pass가 되어 m이 커질수록 false positive를 유발하는데, DTS는 이 느슨한 LBS를 엄격한 Comp UBS로 대체한다. 최종 점수는 ScoreDTS=Σ(UBSscore_j+CompUBSscore_j)(Eq.4)로 산출된다.

PDF p. 4, 5, 6

구성요소 B: page-buffer deficit accumulator (ΔScoreDTS)

각 m-cell group에서 나온 1-bit DTS 결과는 SA(sense amplifier)→SL→CL 경로를 거쳐 기존 QLC 4-bit data latch(DL)에 누적된다. 이때 절대 점수 대신 window-level 최댓값 Smax 대비 부족분인 deficit ΔScoreDTS=Smax-S를 계산하는데, 이 표현 덕분에 per-BL로 넓은 추가 저장공간 없이 4-bit 이내의 compact score를 유지할 수 있다. accumulator는 256개 BL마다 1개를 공유하는 시분할 방식으로 배치되어 면적 오버헤드를 최소화한다.

PDF p. 5, 6

구성요소 C: stage-aware m-adaptive sensing

단계별로 병렬 센싱하는 wordline 수(m1, m2, m3)를 다르게 설정하는 아키텍처 knob이다. 정확도에 민감한 centroid 선택(1단계)과 재순위화(3단계)에는 작은 m(예: 1–4)을, 처리량이 중요한 coarse Top-K2 검색(2단계)에는 큰 m(예: 4–8)을 적용한다. 논문 표기 규칙상 DTS111은 m=(1,1,1), DTS242는 m=(2,4,2)를 의미하며, 더 공격적인 변형으로 D-NOVA-F(DTS282), D-NOVA-XF(DTS484)가 있다.

구성요소 D: contrastive adapter

nn.LayerNorm → nn.Linear → 활성화(GELU) → nn.Linear로 구성된 경량 query-side adapter다. DTS 검색 결과에서 채굴한 hard negative를 사용하는 retrieval-aware InfoNCE loss로 offline에서 한 번 학습된다. 인코더(all-MiniLM-L6-v2, all-mpnet-base-v2)의 출력을 DTS-정렬 표현으로 재구성하여 recall 저하를 완화한다. 중요한 설계 선택은 passage 임베딩은 3D NAND에 고정한 채 query 측에만 적용한다는 점으로, 이 덕분에 이미 기록된 대규모 벡터를 다시 쓰지 않아도 된다. 384-D 임베딩 기준 0.4M 파라미터에 불과하며 query encoding 시간에 0.5–1%의 오버헤드만 추가한다.

구성요소 E: locality-aware mapping

centroid 벡터들 사이를 greedy nearest-neighbor walk로 순회하여 similarity chain을 구성한 뒤, 그 순서를 따라 first-fit 규칙으로 IVF list를 sub-block에 배치한다. 함께 접근되는(co-access) cluster들이 인접 sub-block에 모이므로 질의당 활성화해야 하는 sub-block 수가 줄어든다.

1단계: Centroid search

centroid 데이터는 각 plane의 첫 sub-block에 분산 저장된다. SSD 컨트롤러가 브로드캐스트한 query가 WL driver를 통해 wordline 전압으로 인가되고, 작은 m(1–4)으로 DTS 기반 centroid 매칭이 수행된다. segment별 1-bit 결과가 SA→SL→CL을 거쳐 page buffer DL에 ΔScoreDTS로 누적되고, 여러 plane의 점수가 SSD 컨트롤러에서 병합되어 centroid당 6-bit ΔScoreDTS와 경량 metadata({CID, DIST, CADR}, 약 37-bit)가 생성된다. 이후 embedded processor의 quickselect로 상위 Np(예: 64)개 cluster가 선택되며, SRAM footprint는 0.5KB 미만이다.

PDF p. 6, 7

2단계: Coarse Top-K2 search

선택된 cluster에 속한 벡터만 평가하여 coarse Top-K2 후보 리스트를 만든다. CID는 cluster directory(lookup table)를 통해 물리적 NAND 주소로 변환되고, 해당 sub-block들이 더 큰 m(4–8)으로 DTS 비교를 수행한다. 부분 점수는 로컬에서 누적된 뒤 컨트롤러에서 병합되며, embedded processor가 quickselect로 Top-K2({EID, EADR})를 식별한다. 이 단계는 데이터 볼륨이 가장 크므로 큰 m으로 처리량을 우선한다.

PDF p. 7

3단계: Fine Top-K1 re-ranking 및 문서 반환

Top-K2에 해당하는 소수의 plane/sub-block만 재활성화하여 작은 m(1–4)으로 정밀 재센싱을 수행하고, 누적 후 embedded processor가 최종 quicksort로 Top-K1 결과를 산출한다. 이 단계는 전체 지연시간·에너지의 0.1% 미만만 차지한다 — REIS에서 전체 검색 시간의 약 70%를 차지하던 재순위화가 사실상 사라진 것이 핵심 성과다. 이후 Top-K 식별자가 로컬에서 document 주소로 변환되어 해당 텍스트가 document region에서 직접 fetch되고, 결과 문서만 host로 전송된다.

DTS의 핵심 수식

UBS(Eq.1)는 VWL=qi+α로 인가했을 때 m-cell group의 모든 cell이 ri<qi+α를 만족해야 pass한다. LBS(Eq.2)는 group 내 하나라도 조건을 만족하면 pass하는 느슨한 조건이라 false positive를 만든다. Comp UBS(Eq.3)는 4-bit complement 값 ri'=15-ri, qi'=15-qi에 대해 UBS와 동일한 all-pass 조건을 적용함으로써 반대 방향의 bound를 엄격하게 잡는다. 최종 점수는 ScoreDTS=Σ_j(UBSscore_j + CompUBSscore_j)(Eq.4)이다. 두 방향 bound를 모두 엄격한 all-pass로 구성했기 때문에 후보 집합이 tight하게 좁혀진다.

PDF p. 4, 5, 6

센싱 사이클 절감 효과

DTS의 센싱 시간은 TDTS=(D/m)×2로, 차원 수 D를 m개씩 묶어 처리하고 UBS/Comp UBS 2회를 수행한다. 반면 전형적인 QLC read는 TQLC=D×Nrd로 각 차원마다 여러 번의 read reference를 거쳐야 한다. 이 차이로 DTS는 QLC 전형적 read 대비 sensing cycle을 최대 9.3배 줄이며, 추가 저장공간 오버헤드 없이 REIS 대비 상응하는 speedup을 얻는다.

PDF p. 6

Adapter 학습 알고리즘

adapter는 retrieval-aware InfoNCE loss로 학습된다. 중요한 점은 negative sample을 무작위가 아니라 실제 DTS 검색 결과에서 채굴한 hard negative로 삼는다는 것이다. 즉 DTS가 실제로 혼동하는 사례를 직접 학습 신호로 사용하여 DTS 도메인에서의 순위를 교정한다. 학습은 base database에 대해 offline에서 한 번만 수행되고, 이후 검색 시에는 frozen 상태로 재사용된다.

이 논문의 본질적 기여는 '더 빠른 검색 회로'가 아니라 '거리 척도를 하드웨어에 맞춰 다시 정의하고, 그로 인한 정확도 손실을 학습으로 되사오는(buy back) 설계 패턴'이다. NAND의 직렬 string 구조가 자연스럽게 all-pass 논리를 제공한다는 물리적 사실을 거리 척도로 승격시킨 것이 DTS이고, 그렇게 만들어진 비표준 척도와 표준 임베딩 공간 사이의 간극을 offline contrastive adapter가 메운다. 특히 adapter를 query 측에만 적용해 이미 NAND에 기록된 passage 임베딩을 건드리지 않도록 한 선택은, 대규모 벡터 DB의 재기록 비용을 고려하면 실용성 측면에서 중요한 결정으로 보인다(analyst_inference).

논문의 강점

  • 분석자 해석 하드웨어 물리 특성(NAND string의 직렬 on/off)을 회피 대상이 아니라 거리 척도의 설계 원리로 승격시킨 점이 이 논문의 가장 독창적인 부분이다. 아날로그 crossbar 누적을 NAND에 억지로 이식하는 대신, all-pass 논리를 그대로 유사도 정의에 사용하여 완전 digital 동작과 non-ideality 회피를 동시에 얻었다.
    PDF p. 2, 4, 5
  • 분석자 해석 비표준 거리 척도 도입에 따른 정확도 손실을 하드웨어를 복잡하게 만들어 해결하지 않고, 0.4M 파라미터짜리 offline adapter라는 소프트웨어 수단으로 되사온 설계 분업이 명확하다. 검색 시점 오버헤드가 query encoding의 0.5–1%에 그친다는 점에서 비용 배분이 합리적이다.
    PDF p. 7, 8
  • 분석자 해석 adapter를 query 측에만 적용하고 passage 임베딩은 NAND에 고정한 선택은, 수백만~수천만 문서 규모의 벡터를 재기록해야 하는 비용을 회피한다는 점에서 실제 배포 관점의 현실성이 높다.
    PDF p. 8
  • 분석자 해석 검증이 다층적이다. 알고리즘 수준(6개 데이터셋 × 2개 인코더 recall), 회로 수준(ASAP7 7nm 합성, in-house DTS circuit simulator), 신뢰성 수준(QLC VTH 잡음에 대한 σ당 1,000회 Monte Carlo)을 모두 제시하여 cross-layer 주장의 각 층을 개별적으로 뒷받침한다.

핵심 용어

RAG (Retrieval-Augmented Generation)
LLM이 사전학습 지식만 사용하는 대신 외부 지식 베이스에서 관련 문서를 검색해 함께 입력하는 기법. 지식 최신화가 쉽지만 검색 단계가 시스템 비용을 지배하게 된다. PDF p. 1
DTS (Dual-bound Tight Similarity Sensing)
이 논문이 제안한 NAND 전용 거리 척도. UBS와 Complement UBS라는 두 개의 엄격한 all-pass 조건을 결합해 상한·하한을 동시에 좁히며, MLC 반복 read 없이 binary sensing만으로 유사도를 평가한다(Eq.1–4). PDF p. 2, 4, 5, 6
UBS (Upper-Bound Similarity Sensing)
wordline 전압을 VWL=qi+α로 인가하고, m-cell group 내 모든 cell이 ri<qi+α를 만족할 때만 pass로 판정하는 센싱(Eq.1). NAND string의 직렬 구조가 all-pass 논리를 자연스럽게 구현한다. PDF p. 4, 5
Comp UBS (Complement UBS)
4비트 complement 값(ri'=15-ri, qi'=15-qi)에 UBS와 동일한 all-pass 조건을 적용하는 센싱(Eq.3). 느슨한 LBS를 대체하여 false positive를 억제한다. PDF p. 5, 6
LBS (Lower-Bound Sensing)
m-cell group 내 단 하나의 cell만 조건을 만족해도 pass로 판정하는 기존의 느슨한 센싱(Eq.2). m이 커질수록 false positive가 급증하여 DTS에서 Comp UBS로 대체되었다. PDF p. 5, 6, 11
α (tolerance margin)
wordline 전압에 더해지는 센싱 여유 마진. QLC의 VTH 잡음에 대한 강건성을 결정하며, α=1은 잡음 영역에서 붕괴하고 α=2가 nominal 동작점으로 채택되었다. PDF p. 11
m (parallel-sensed wordline 수)
block당 동시에 센싱되는 wordline 개수이자 정확도-처리량 절충의 핵심 knob. 논문 표기 DTS111은 m=(1,1,1), DTS242는 m=(2,4,2)를 뜻하며 D-NOVA-F는 DTS282, D-NOVA-XF는 DTS484이다. PDF p. 7, 11, 12
ΔScoreDTS (deficit score)
window-level 최댓값 Smax 대비 부족분(Smax-S)으로 점수를 표현하는 방식. per-BL 추가 저장공간 없이 기존 QLC 4-bit data latch에 compact하게 누적할 수 있게 한다. PDF p. 5, 6
IVF (Inverted File Index)
벡터 공간을 Nnlist개 cluster로 분할하고 질의 시 Nnprobe개 cluster만 탐색하는 ANN 인덱스(예: 1024 → 64). 접근 패턴이 구조적이어서 NAND의 block/plane 병렬성에 매핑하기 좋다. PDF p. 1, 7
Contrastive adapter
LayerNorm-Linear-GELU-Linear로 구성된 경량 query-side 투영 모듈. DTS 검색에서 채굴한 hard negative 기반 InfoNCE loss로 offline에서 한 번만 학습되어 임베딩을 DTS-정렬 공간으로 옮긴다. PDF p. 7, 8
QLC (Quad-Level Cell)
cell당 4비트(16개 상태)를 저장하는 NAND 방식. D-NOVA는 임베딩을 INT4로 각 cell에 인코딩하며, 상태가 조밀해 VTH 잡음에 취약하다는 점이 α 설계의 배경이다. PDF p. 4, 11
Locality-aware mapping
centroid 간 greedy nearest-neighbor walk로 similarity chain을 만든 뒤 first-fit으로 IVF list를 sub-block에 배치하여, co-access되는 cluster를 인접 배치해 질의당 sub-block 활성화 수를 줄이는 기법. PDF p. 8, 12
REIS
비교 대상 SOTA in-storage 가속기. NAND 내부에서 binary quantization 기반 coarse search를 수행하지만, 정확도 보완을 위한 컨트롤러 측 INT8 re-ranking이 전체 검색 시간의 약 70%를 차지한다. PDF p. 2, 3, 9
NSP (Near-Storage Processor)
RAGX 유사 비교 baseline. QLC NAND에 INT8 임베딩을 저장한 뒤 일반 read로 fetch하고 별도 ASIC이 cosine similarity를 계산하는 방식으로, 배열 외부 처리에 의존한다. PDF p. 9
Recall@K gap
FP32-IVF(software 상한) 대비 Recall@K의 차이로 정의되는 정확도 손실 지표. 이 논문의 주 정확도 척도는 Recall@25와 Recall@100이다. PDF p. 8, 10
Page buffer / SA-SL-CL
NAND 주변 회로 경로. sense amplifier(SA)에서 감지된 1-bit DTS 결과가 SL, CL을 거쳐 data latch(DL)로 전달·누적되며, D-NOVA는 이 기존 경로를 재사용해 하드웨어 오버헤드를 최소화한다. PDF p. 5, 6
