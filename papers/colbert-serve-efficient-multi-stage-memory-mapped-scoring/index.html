ColBERT-serve: Efficient Multi-Stage Memory-Mapped Scoring

저자 · Kaili Huang, Thejas Venkatesh, Uma Dingankar, Antonio Mallia, Daniel Campos, Jian Jiao, Christopher Potts, Matei Zaharia, Kwabena Boahen, Omar Khattab, Saarthak Sarup, Keshav Santhanam 출처 · ECIR 2025 (Advances in Information Retrieval, Lecture Notes in Computer Science, vol 15575) 연도 · 2025

Index

핵심 요약

ColBERT과 같은 multi-vector late-interaction retriever는 높은 검색 품질을 보이지만, PLAID로 인코딩된 ColBERTv2 인덱스를 완전히 메모리에 올리려면 Wikipedia 규모(2000만 passage)에서 거의 100GB의 RAM이 필요해, 저사양·저비용 서버에서 다수의 동시 사용자를 서빙하기 어렵다는 문제가 있다. 이 논문은 ColBERT-serve라는 서빙 시스템을 제안하는데, (1) 압축된 ColBERTv2 임베딩 텐서에 memory-mapping을 적용해 인덱스 대부분을 디스크에 두고 필요한 페이지만 OS가 온디맨드로 메모리에 올리도록 하며, (2) SPLADEv2를 1단계 검색기로 사용하는 multi-stage retrieval로 memory-mapped 인덱스에 대한 접근(페이지 미스)을 최소화하고, (3) SPLADEv2와 ColBERTv2 점수를 z-norm 정규화 후 선형 보간(hybrid scoring)하여 재랭킹만 사용할 때보다도 품질을 개선하며, (4) Python GIL 해제와 서버-클라이언트 구조로 동시 요청을 병렬 처리한다. 그 결과, MS MARCO와 Wikipedia에서 모델 로딩에 필요한 RAM을 각각 90%, 92% 절감하고 머신 비용을 78%, 68% 절감하면서도, MS MARCO Dev/Test와 LoTTE Lifestyle에서는 완전 인메모리 ColBERTv2보다 오히려 높은 품질을 달성했고, 초당 최대 4~5개의 쿼리를 낮은 지연시간으로 서빙할 수 있음을 실증했다.

    PDF p. 1, 5, 6
  • 논문 주장 SPLADEv2 1단계 검색 + ColBERTv2 재랭킹의 multi-stage 구조로 memory-mapped 인덱스의 페이지 미스로 인한 지연시간 증가를 완화했다.

    PDF p. 4, 5

    PDF p. 5, 6

해결하려는 문제

논문 주장

고동시성·저메모리 환경에서의 late-interaction 검색 서빙

ColBERT, ColPali와 같은 multi-vector late-interaction retriever는 최신 수준의 품질과 일반화 성능을 낮은 지연시간으로 달성한다. 그러나 임베딩 압축 기술(PLAID 등)의 발전에도 불구하고, Wikipedia 규모(2000만 passage)의 ColBERTv2 인덱스를 PLAID로 서빙하려면 거의 100GB의 RAM이 필요하다. 이는 RAM이 적은 저가 서버에서 다수의 동시 사용자를 낮은 지연시간으로 서빙해야 하는 상황에서 큰 제약이 되며, 기존 신경망 IR 문헌에서는 이러한 고동시성·저메모리 환경에서의 비용·지연시간·품질 간 트레이드오프가 거의 다뤄지지 않았다.

기존 방식의 한계

논문 주장

메모리 매핑 기반 접근의 선행 연구

Memory-mapping은 근사 최근접 이웃 검색이나 일부 데이터베이스 시스템에서 사용되어 왔고, 동시대 연구([18])는 신경망 IR에도 이를 적용해 SSD 지연의 영향을 줄이는 prefetcher를 구축했다. 그러나 본 논문은 prefetching 대신, multi-stage retrieval을 통해 디스크 접근 횟수 자체를 줄이는 방향을 택한다는 점에서 차별화된다.

PDF p. 2

재랭킹 기반 접근의 품질 손실

[7]은 BM25가 생성한 top-1000 결과를 ColBERT로 재랭킹하는 방식으로 지연시간은 개선했으나 MRR과 recall에서 상당한 성능 저하를 겪었다. [13]도 ColBERTv2로 BM25 후보를 재랭킹하고 LADR을 결합해 더 효과적인 시스템을 만들었지만, 두 시스템 모두 MS MARCO에서 품질 손실을 피하지 못했다. 즉 기존의 재랭킹 기반 접근은 지연시간·비용을 낮추는 대신 품질을 희생하는 트레이드오프에 머물러 있었다.

핵심 contribution

평가 방법론 및 벤치마크

다양한 트래픽 워크로드와 메모리 예산 하에서 신경망 IR 모델의 동시 서빙(concurrent serving)을 평가하기 위한 방법론과 벤치마크를 제시한다.

PDF p. 2

ColBERT-serve 시스템

ColBERT-serve를 도입한다: (1) ColBERTv2 인덱스 대부분을 디스크에 유지할 수 있는 새로운 memory-mapping 아키텍처, (2) multi-stage retrieval을 통한 인덱스 접근 최소화, (3) PISA와 ColBERTv2를 개조해 동시 요청을 병렬로 낮은 지연시간에 처리하고 부하에 따라 원활히 확장, (4) hybrid scoring을 통한 전체 ColBERTv2 검색 품질 보존.

PDF p. 2

실증 평가

대규모 컬렉션에 대해 품질을 유지하면서 단 몇 GB의 RAM만으로 초당 최대 4개의 쿼리를 서빙할 수 있는 최초의 ColBERT 서빙 시스템임을 입증했다(전체 ColBERTv2 대비 모델 로딩 RAM 사용량 90% 감소).

PDF p. 1, 2

제안 방식의 전체 구조

논문 주장

전체 구성

ColBERT-serve는 압축된 ColBERTv2 임베딩 텐서에 memory-mapping을 적용해 인덱스를 디스크에 상주시키고, SPLADEv2 기반 1단계 검색과 ColBERTv2 재랭킹을 결합한 multi-stage 아키텍처로 인덱스 접근을 최소화하며, 두 단계의 점수를 결합하는 hybrid scoring으로 품질을 보존하고, 서버-클라이언트 구조와 Python GIL 해제를 통해 동시 요청을 병렬 처리하는 네 요소로 구성된다.

PDF p. 2, 3, 4

단계별 동작 과정

논문 주장

1단계: SPLADEv2 + PISA 후보 검색

SPLADEv2(sparse lexical and expansion model)가 쿼리를 확장하고, PISA 엔진이 효율적으로 후보 문서 top-200개를 검색한다(SPLADEv2 w/ PISA).

PDF p. 3

2단계: MMAP ColBERTv2 재랭킹

memory-mapped ColBERTv2 인덱스(MMAP ColBERTv2)가 top-200 후보들만을 대상으로 late-interaction 재랭킹을 수행한다(MMAP Rerank). 후보를 200개로 좁힘으로써 디스크 페이지 미스가 발생하는 인덱스 접근 범위를 크게 줄인다.

PDF p. 3

3단계: Hybrid scoring

SPLADEv2 점수와 MMAP ColBERTv2 점수를 각각 z-norm으로 정규화한 뒤, 계수 α를 이용한 선형 보간으로 결합하여 최종 hybrid score를 산출한다(MMAP Hybrid). 수식은 Shybrid(D,Q) = αN(SSPLADE(D,Q)) + (1-α)N(SColBERT(D,Q)), N(x) = (x-x̄)/S 이다.

PDF p. 3, 4

핵심 알고리즘 또는 인덱스 구조

논문 주장

Memory-Mapped Storage

압축된 ColBERTv2 임베딩을 인코딩하는 텐서에 memory-mapping을 도입해, 인덱스를 사전에 전부 로딩하지 않고 OS가 페이지 단위(page granularity)로 필요한 데이터만 메모리에 올리며 RAM이 부족하면 페이지를 축출(evict)하도록 한다. 이를 통해 RAM 요구량을 90% 이상 절감한다.

PDF p. 2, 3

Concurrent Requests (서버-클라이언트 구조)

ColBERTv2의 C++ 확장 호출 시 Python GIL을 해제하여 멀티스레딩 호환성을 개선하는데, 이것이 없으면 각 쿼리가 확장 호출 시 블록되어 멀티스레딩이 사실상 불가능하고, 프로세스 다중화로만 동시성을 확보해야 하며 memory-mapping 없이는 프로세스별 메모리가 선형으로 증가한다. 저자들은 스레드 수를 튜닝한 결과, 낮은 부하에서는 멀티스레딩이 유리하지만 높은 부하에서는 단일 스레드 성능이 더 우수함을 발견하여 모든 실험에서 단일 스레드를 사용했다. PISA 엔진도 이 서버-클라이언트 구조와 multi-stage retrieval에 맞게 개조되었다.

PDF p. 2, 3

Multi-Stage Retrieval

memory-mapped 인덱스는 페이지 미스로 인해 MS MARCO에서 인메모리 인덱스 대비 검색이 약 2배 느려지는 문제를 안고 있다. 이를 SPLADEv2를 1단계 검색기로 사용해 ColBERT 인덱스에 접근해야 하는 문서 수를 top-200으로 최소화함으로써 완화한다.

PDF p. 3

Hybrid Scoring 정규화 방식 선택

SPLADEv2와 ColBERTv2는 점수 분포가 크게 다르므로, 이를 결합하기 위해 (1) [0,1] 선형 매핑, (2) min-max norm, (3) z-norm 세 가지를 검토했고 z-norm이 가장 좋은 결과를 보여 최종 정규화 방식으로 채택했다.

PDF p. 3, 4

실험 구성

논문 주장

데이터셋

MS MARCO Passage Ranking Dev(7K 쿼리, 8.8M passage, in-domain)를 기본 벤치마크로 사용하고, ColBERT 저자들이 사용한 held-out 5K Test 셋으로 평가 결과를 보고한다. Out-of-domain(OOD) 일반화 검증을 위해 Wikipedia Open-QA NQ-dev(8.7K 쿼리, 21M passage)와 LoTTE Search Lifestyle-dev(417 쿼리, 269K passage)를 사용한다. Wikipedia는 RAM 사용량에 부담을 주는 대규모 컬렉션인 반면 LoTTE Lifestyle은 항상 메모리에 들어가는 소규모 컬렉션으로, 서로 대비되는 조건을 제공한다.

PDF p. 4

베이스라인과 지표

비교 대상은 ColBERTv2(전체 인메모리 PLAID), SPLADEv2(BT-SPLADE-L 체크포인트 + PISA), MMAP ColBERTv2, MMAP Rerank(SPLADEv2 top-200 후 재랭킹), MMAP Hybrid(선형 보간)이다. 품질 지표는 MRR@10, Recall@5, Recall@50, Success@5를 사용하고, 효율성은 P95 latency, RSS 기준 RAM 사용량, AWS 월 비용($/month)으로 측정한다. MS MARCO·Wikipedia는 r6a.4xlarge(전체 ColBERTv2 대조군), m5ad.xlarge/r6id.xlarge(MMAP 실험) 인스턴스를, LoTTE는 c5ad.xlarge 인스턴스를 사용했다.

PDF p. 4

주요 결과와 의미

논문 주장

검색 품질

MS MARCO Dev에서 Hybrid(α=0.3)가 모든 방법 중 가장 우수했다(MRR@10 40.22, R@5 57.38, R@50 86.98). MS MARCO 5K Test에서도 Hybrid(α=0.3)가 완전 인메모리 ColBERTv2(MRR@10 40.57)보다 높은 MRR@10 41.11을 기록했다. 반면 Wikipedia(OOD)에서는 MS MARCO로 튜닝한 α=0.3의 Hybrid(S@5 65.78, -2.6%)가 Rerank(S@5 66.29, -1.8%)보다 낮았으나, 데이터셋별 최적 α를 사용하면 Hybrid(S@5 66.34, -1.7%)가 Rerank를 다시 앞질렀다. LoTTE Lifestyle에서는 Hybrid(최적 α)가 S@5 75.3(+0.9%)으로 완전 인메모리 ColBERTv2(S@5 74.6)를 능가했다.

PDF p. 4, 5

RAM 사용량 및 비용

memory-mapping 적용 시 ColBERTv2 모델 로딩에 필요한 RAM(RSS 기준)이 MS MARCO에서 23.4GB→2.3GB(90% 감소), Wikipedia에서 98.3GB→8.2GB(92% 감소)로 크게 줄었다. 이에 따라 저용량 메모리 머신에서 인덱스를 호스팅할 수 있게 되어 AWS 월 비용이 MS MARCO 78%, Wikipedia 68% 감소했다.

PDF p. 4, 5, 6

트래픽 부하에 따른 지연시간

Wikipedia에서 PISA 기반 SPLADEv2가 가장 낮은 지연시간을 보였으나 품질(특히 OOD)에서 큰 손실이 있었다. Rerank/Hybrid는 SPLADE보다 느리지만 memory-mapped ColBERTv2 단독보다는 훨씬 빠르며, QPS 5(1/0.2)까지는 낮은 지연시간을 유지하다가 그 이상에서 큐잉으로 인해 지연시간이 급격히 증가했다. 더 고성능·고비용 머신에서 실행된(참고용) 완전 인메모리 ColBERTv2조차 QPS 3.3(1/0.3) 미만 구간에서 Rerank/Hybrid보다 지연시간이 높았다. MS MARCO에서도 Rerank/Hybrid가 모든 트래픽 부하 구간에서 memory-mapped ColBERTv2의 지연시간을 크게 줄였고(단, 완전 ColBERTv2와는 상이한 프로세서 머신에서 평가되어 직접 비교는 어려움), LoTTE에서도 유사한 패턴이 관찰되었다.

PDF p. 6, 7

결론

논문 주장

요약

저자들은 memory-mapping, hybrid scoring, 동시 요청 지원을 결합한 서빙 시스템 ColBERT-serve와, 고동시성·메모리 제약 환경에서 신경망 IR 트레이드오프를 평가하는 방법론을 제시했다. 그 결과 단 몇 GB의 RAM만으로 초당 여러 쿼리를 서빙할 수 있는 최초의 ColBERT 서빙 시스템임을 입증했다고 결론짓는다.

기존 방식과 제안 방식 비교

논문의 강점

  • 분석자 해석 memory-mapping과 multi-stage retrieval을 결합해 RAM 절감과 지연시간 감소를 동시에 달성하면서도 여러 데이터셋에서 완전 인메모리 ColBERTv2와 동등하거나 더 나은 품질을 보였다는 점에서, 기존 재랭킹 방식들이 겪던 '효율 대 품질'의 트레이드오프를 상당 부분 해소했다.
    PDF p. 4, 5, 6
  • 분석자 해석 in-domain(MS MARCO)뿐 아니라 OOD 데이터셋(Wikipedia, LoTTE)까지 포함한 평가를 통해 일반화 가능성을 함께 점검했다.
    PDF p. 4, 5
  • 분석자 해석 RAM, 지연시간, 비용, 품질을 모두 정량화하고 이를 실제 AWS 인스턴스 사양·비용에 매핑해, 실무 배포 관점에서 바로 활용 가능한 근거를 제시했다.
    PDF p. 4, 5, 6
  • 분석자 해석 PISA와 ColBERTv2 구현 자체를 GIL 해제, 서버-클라이언트 구조로 개조해 시스템 수준의 엔지니어링 기여를 포함하고 있다.
    PDF p. 2, 3

한계

  • 저자 명시 한계 Wikipedia(OOD)에서 MS MARCO Dev로 튜닝한 α=0.3의 Hybrid는 Rerank보다 낮은 성능을 보였는데, 이는 OOD 환경에서 held-out set으로 α를 별도 튜닝하는 것이 중요할 수 있음을 시사하며, 저자들은 이를 향후 연구 과제로 남겨두었다.
    PDF p. 5
  • 저자 명시 한계 MS MARCO 및 Wikipedia 실험에서 전체 ColBERTv2 베이스라인은 인덱스를 RAM에 담을 수 있는 더 고사양·고비용 머신(그리고 다른 물리적 프로세서)에서 평가되었기 때문에, 그 지연시간은 참고용일 뿐 memory-mapped 방법들과 직접 비교할 수 없다고 저자들이 명시했다.
    PDF p. 4, 6, 7

핵심 용어

Late-interaction retrieval
쿼리와 문서를 각각 토큰 단위의 다중 벡터(multi-vector)로 인코딩한 뒤, 검색 시점에 토큰 벡터 간 유사도를 계산해 결합하는 방식. ColBERT가 대표적이다. PDF p. 1
PLAID
ColBERTv2의 압축된 임베딩 인덱스를 효율적으로 검색하기 위한 인덱싱/스코어링 기법. PDF p. 1, 3
Memory-mapping
디스크에 있는 데이터를 가상 메모리 주소 공간에 매핑해, 실제로 접근된 부분만 온디맨드로 물리 메모리에 올리고 필요 시 페이지를 축출하는 OS 수준 기법. PDF p. 2, 3
SPLADEv2
sparse lexical and expansion model. 쿼리와 문서를 확장된 어휘 기반 sparse 벡터로 표현하는 1단계 검색 모델로, 본 논문에서는 PISA 엔진과 함께 사용된다. PDF p. 3
PISA
sparse retrieval을 위한 고성능 인덱싱/검색 엔진으로, 본 논문에서 SPLADEv2 기반 1단계 검색을 수행하는 데 사용되고 서버-클라이언트 구조에 맞게 개조되었다. PDF p. 2, 3
Hybrid scoring
SPLADEv2 점수와 ColBERTv2 점수를 z-norm으로 정규화한 뒤 계수 α로 선형 보간하여 결합하는 방식. PDF p. 3, 4
z-norm
값에서 평균을 빼고 표준편차로 나누는 정규화 방식(N(x) = (x-x̄)/S). 본 논문에서 SPLADEv2와 ColBERTv2의 서로 다른 점수 분포를 결합하기 위해 채택되었다. PDF p. 3, 4
RSS (Resident Set Size)
프로세스가 실제로 점유하고 있는 물리 메모리 크기를 나타내는 지표로, 본 논문에서 RAM 사용량 측정 기준으로 사용된다. PDF p. 5
P95 latency
요청 지연시간 분포에서 95번째 백분위수에 해당하는 값으로, 서버 부하 하에서의 체감 성능을 나타내는 지표. PDF p. 4, 6
