MM-Matryoshka: Towards Budget-Elastic Visual Document Retrieval via a 2D Multimodal Matryoshka Training Framework

저자 · Haowen Xiang, Yibo Yan, Jiahao Huo, Yu Huang, Yi Cao, Mingdong Ou, Xuming Hu 출처 · Venue 확인 필요 연도 · 2026

핵심 요약

MM-Matryoshka는 ColPali 계열의 multi-vector Visual Document Retrieval(VDR) 리트리버가 배포 시 겪는 저장소·연산 비용 문제를, vector width(임베딩 차원)와 encoder depth(transformer layer 깊이)를 동시에 탄력적으로 만드는 2D Matryoshka training framework로 해결한다. 학습은 두 개의 목적함수로 구성된다. Dimension-level MRL loss(Lprefix)는 final layer에서 여러 prefix 차원 d가 각각 곧바로 late interaction에 사용될 수 있도록 in-batch cross-entropy로 학습시키고, Layer-level MRL loss(Ldistill)는 final layer의 in-batch score 분포를 teacher(stop-gradient)로 삼아 중간 layer(student)로 KL divergence 기반 score distillation을 수행한다. 두 손실은 강도 λ로 결합되어 단일 모델이 backbone별로 구성된(configured) 2D budget grid S×D 전체를 커버하도록 만든다. 추론 시에는 추가 학습 없이 이 grid 안에서 원하는 budget (ℓ,d)를 골라 해당 layer의 token vector를 뽑고 prefix truncation 후 재정규화한 뒤, 학습 때와 동일한 MaxSim late-interaction 함수로 점수를 매긴다. PaliGemma, Qwen2-VL, Qwen2.5-VL 세 backbone과 ViDoRe v1·v2의 14개 데이터셋에서 검증했으며, Qwen2.5-VL 기반 모델은 middle layer·32차원 budget에서 ViDoRe v1 retention 97.6%를 기록해 같은 조건의 ColQwen2.5 baseline 47.8%를 크게 앞선다.

해결하려는 문제

논문 주장

multi-vector VDR의 구조적 비용

Visual Document Retrieval(VDR)은 자연어 질의에 대해 관련 문서 페이지를 검색하되, 텍스트뿐 아니라 layout, 표, 차트 등 시각적 증거를 보존해야 하는 과제이다. ColPali류의 multi-vector 리트리버는 late interaction(MaxSim)을 수행하기 위해 페이지마다 다수의 token vector를 encoding하고 저장한다. MaxSim의 연산 비용은 query vector 수, page vector 수, embedding 차원 수에 비례해 증가하므로, 실제 배포에서는 저장소와 연산 overhead가 모두 큰 병목이 된다.

PDF p. 1, 2

encoder depth라는 추가 비용 축

최신 VLM 기반 encoder는 깊이가 깊어서 encoder depth 자체도 무시할 수 없는 비용 요인이 된다. 따라서 실용적인 VDR 리트리버는 높은 full-budget accuracy만으로 충분하지 않고, 저장소와 latency를 위해 정확도를 의도적으로 교환할 수 있는 controllable budget을 함께 제공해야 한다.

PDF p. 1

논문이 던지는 핵심 질문

논문이 다루는 효율성 질문은 '다수 벡터를 의도적으로 유지하는 multi-vector VDR 시스템에서, 동일한 하나의 리트리버가 더 낮은 비용의 selectable budget을 노출할 수 있는가'이다. 즉 벡터 수를 줄이는 것이 아니라, 같은 모델이 여러 비용 지점에서 동작하도록 만드는 것이 목표이다.

PDF p. 1, 2

기존 방식의 한계

논문 주장

텍스트 기반 MRL: vector width만 제어

전통적인 Matryoshka Representation Learning(MRL) 설정은 임베딩 차원, 즉 vector width만 통제 대상으로 삼으며 encoder depth는 budget 공간 밖에 남겨둔다. 결과적으로 비용의 한 축만 조절 가능하다.

PDF p. 1

Vision 기반 MRL: 대상이 다름

Matryoshka Multimodal Models(Cai et al. 2025) 같은 vision 기반 MRL은 LLM에 전달되는 visual token의 coarse-to-fine 세부 수준을 제어할 뿐, retrieval-vector width나 encoder depth를 제어하지 않는다. 또한 Figure 1(b)에 라벨된 대로 layout 및 구조적 관계를 보존하지 못한다.

PDF p. 1, 3

Token pruning/merging: sequence 축만 축소

DocPruner 등 pruning 기반, Light-ColPali·ColChunk 등 merging 기반, 그리고 학습형 요약 토큰이나 attention 기반 hybrid 기법은 주로 sequence 차원의 token 개수를 줄인다. 그러나 리트리버가 VLM encoder의 final-layer representation에 의존한다는 근본적 비용 요인은 그대로 남으며, 고정된 representation scale에서만 동작한다.

PDF p. 2, 3

고정확도 multi-vector 시스템의 저장소 병목

Llama-Nemoretriever-Colembed, Nemotron ColEmbed V2와 같은 최근 multi-vector 시스템은 top-tier 정확도를 유지하지만, 문서당 수천 개의 고차원 임베딩을 요구해 저장소 병목을 오히려 심화시킨다.

PDF p. 2

기존 2D Matryoshka 연구와의 간극

2D Matryoshka sentence embeddings 등 기존 2D 연구는 대부분 text retrieval이나 일반적인 단일 벡터 representation learning을 대상으로 하며, late interaction에서 요구되는 다수의 vector-to-region alignment를 보존해야 한다는 multi-vector VDR 고유의 제약을 다루지 않는다.

PDF p. 3

핵심 contribution

논문 주장

VDR을 위한 최초의 2D training framework

VDR을 위한 2D training framework인 MM-Matryoshka를 최초로 formulate하여, 단일 리트리버가 vector width와 encoder depth 양쪽 모두에서 효율성과 정확도의 균형을 맞출 수 있도록 했다.

PDF p. 2

backbone 간 일반성

MM-Matryoshka는 PaliGemma, Qwen2-VL, Qwen2.5-VL 등 대표적인 여러 VLM backbone에 걸쳐 견고하게 작동하며, 개선된 low-dimensional 및 intermediate-layer retention을 통해 기대되는 budget-elastic 동작을 달성한다.

PDF p. 2

포괄적 실험 검증

성능과 효율성을 모두 다루는 14개 VDR 데이터셋에 대한 실험적 검증을 수행하고, 정량적 결과를 정성적 사례 연구(qualitative case study)로 보완한다.

PDF p. 2

제안 방식의 전체 구조

논문 주장

2D budget 공간의 정의

MM-Matryoshka는 vector width(임베딩 prefix 차원 d)와 encoder depth(transformer layer ℓ)를 모두 elastic하게 만든다. budget은 쌍 (ℓ,d)로 정의되며, ℓ=L이고 d=D이면 표준 full-budget 리트리버와 동일하다. 학습이 끝난 뒤에는 별도 모델을 추가로 학습하지 않고, 동일한 단일 모델이 backbone별로 구성된(configured) target layer set S와 prefix dimension set D 안의 (ℓ,d)에서 동작한다.

PDF p. 2, 3, 4

두 축을 담당하는 두 손실

Dimension-level MRL loss(Lprefix)는 final layer에서 여러 prefix 차원이 직접 검색에 쓰일 수 있도록 학습시키고, Layer-level MRL loss(Ldistill, score distillation)는 final layer의 검색 관련 score 분포를 sampled intermediate layer로 전이시킨다. 두 손실이 각각 dimension 축과 depth 축을 담당한다.

PDF p. 3, 4

추론 인터페이스의 불변성

학습과 추론 모두 동일한 MaxSim late-interaction 함수를 사용한다. 추론 시에는 선택한 budget에 해당하는 벡터 prefix를 정규화한 뒤 그대로 MaxSim으로 점수를 계산하므로, 기존 late-interaction 파이프라인과 인터페이스가 바뀌지 않는다.

PDF p. 3, 4

단계별 동작 과정

논문 주장

1단계 — Truncated encoding (Eq.1)

배치 내 query-page 쌍 {(qi,pi)}에 대해, 선택된 layer ℓ까지만 VLM encoder를 실행(f≤ℓ)하여 query/page hidden-state 행렬 Hqℓ∈R^(Tq×h), Hpℓ∈R^(Tp×h)을 생성한다.

PDF p. 3

2단계 — Layer-specific projection (Eq.2)

layer별 projection head ψℓ을 적용하고 토큰 단위 ℓ2 정규화를 수행하여 D차원 query/page retrieval vector Qℓ, Pℓ을 얻는다. ℓ=L이면 표준 full-depth 리트리버, ℓ<L이면 얕은 selectable budget이 된다.

PDF p. 3, 4

3단계 — Prefix truncation (Eq.3)

각 대상 prefix dimension d에 대해 τd로 벡터를 앞에서부터 d차원으로 자르고 재정규화(x1:d/‖x1:d‖2)하여 budgeted representation Qℓ^(d), Pℓ^(d)를 만든다.

PDF p. 3

4단계 — MaxSim 점수 계산 (Eq.4)

budget (ℓ,d) 하에서 query token vector마다 가장 유사한 page token vector와의 내적 최댓값을 구하고 이를 평균하여 query-page 관련성 점수 sℓ,d(q,p)를 산출한다.

PDF p. 3

5단계 — Dimension-level 학습 (Eq.5)

final layer L에서 모든 d∈D에 대해 in-batch score matrix S_L^(d)를 계산하고, 정답 페이지를 positive로 하는 가중치 αd 적용 cross-entropy loss Lprefix로 학습한다.

PDF p. 3, 4

6단계 — Layer-level distillation (Eq.6)

모든 target layer ℓ∈S와 prefix dimension d∈D 조합에 대해, final-layer score matrix(teacher, stop-gradient 적용)와 해당 layer의 score matrix(student) 사이의 KL divergence를 온도 Tdist로 계산하여 Ldistill(ℓ,d)를 구한다.

PDF p. 4

7단계 — Joint loss (Eq.7)

Lprefix와, 강도 λ로 가중되고 그리드 S×D 크기로 정규화된 Ldistill의 합을 결합해 최종 joint loss L을 구성하고 학습한다.

PDF p. 4

8단계 — 추론 시 budget 선택

추론에서는 추가 학습 없이 학습된 2D budget grid(S×D) 내에서 budget (ℓ,d)를 선택하여 해당 layer의 query/page token vector를 추출하고, prefix truncation과 재정규화를 적용한 뒤 동일한 MaxSim 함수로 점수를 매긴다. dimension 축소는 저장소와 dot-product 폭을 직접 줄이고, layer 축소는 이전 layer의 hidden state를 사용하며 early-exit로 구현하면 encoder 연산량도 함께 줄일 수 있다.

PDF p. 4

주요 구성요소와 핵심 알고리즘

논문 주장

Truncated VLM Encoder (f≤ℓ)

선택된 transformer layer ℓ까지만 실행하여 query/page의 hidden-state 행렬 Hqℓ∈R^(Tq×h), Hpℓ∈R^(Tp×h)을 생성한다(Eq.1). backbone-native hidden 차원 h의 구체적 수치는 본문에 기호로만 정의되어 있다.

PDF p. 3

Layer-specific Projection ψℓ + token-wise ℓ2 정규화

layer별 projection과 정규화를 적용해 D차원 retrieval vector Qℓ, Pℓ을 만든다(Eq.2). 각 layer가 자신의 projection head를 갖기 때문에 중간 layer도 독립적인 retrieval 공간을 학습할 수 있다.

PDF p. 3, 4

Prefix Truncation τd

token vector x∈R^D를 prefix dimension d로 잘라 재정규화한다(Eq.3). 재정규화가 있어야 잘린 벡터의 내적이 여전히 코사인 유사도 스케일을 유지한다.

PDF p. 3

MaxSim late-interaction 연산자

budget (ℓ,d)에서 query token vector마다 최대 유사도를 취해 평균하는 표준 late-interaction 점수 함수(Eq.4)로, 학습과 추론에서 동일하게 사용된다. 이 인터페이스 동일성이 budget 전환을 가능하게 하는 핵심 설계이다.

PDF p. 3

Dimension-Level MRL Loss (Lprefix, Eq.5)

final layer L에서 각 prefix dimension d의 in-batch score matrix S_L^(d)에 가중치 αd를 적용한 cross-entropy retrieval loss를 합산한다. 좁은 token vector가 late interaction에서 곧바로 사용 가능해지도록 만드는 역할을 한다.

PDF p. 3, 4

Layer-Level MRL Loss (Ldistill, Eq.6)

final layer를 teacher(stop-gradient)로 고정하고, 각 intermediate/student layer의 score 분포를 온도 Tdist의 KL divergence로 맞춘다. 정답 라벨이 아니라 score 분포 자체를 맞추는 score distillation이므로, 중간 layer가 최종 layer의 검색 결정을 재현하도록 유도된다.

PDF p. 4

Joint Loss (Eq.7)

Lprefix와 그리드 S×D 전체에 대해 평균된 Ldistill을 강도 λ로 결합한다. 두 목적함수를 동시에 최적화하여 하나의 모델이 구성된 2D budget grid(S×D) 전체를 커버한다.

PDF p. 4

Backbone-dependent 2D Budget Grid

각 backbone의 encoder depth, retrieval-head width, 사용 가능한 intermediate hidden state 특성에 따라 target layer set S와 prefix dimension set D를 다르게 선택한다. 동일한 학습 목적함수가 backbone별로 서로 다른 2D trade-off를 다루게 된다.

PDF p. 4

실험 구성

논문 주장

학습 및 평가 데이터

학습에는 HuggingFace의 vidore/colpali_train_set을 사용한다. 평가는 ViDoRe v1 benchmark 10개 sub-dataset(docvqa_test_subsampled, infovqa_test_subsampled, tabfquad_test_subsampled, arxivqa_test_subsampled, tatdqa_test, shiftproject_test, syntheticDocQA의 artificial_intelligence·energy·government_reports·healthcare_industry test)과 ViDoRe v2 benchmark 4개 sub-dataset(esg_reports_v2, biomedical_lectures_v2, economics_reports_v2, esg_reports_human_labeled_v2), 총 14개 VDR 데이터셋에서 수행한다.

PDF p. 5

Backbone과 baseline

backbone은 PaliGemma-3B, Qwen2-VL-2B, Qwen2.5-VL-3B이며, baseline은 각각에 대응하는 public late-interaction 리트리버인 ColPali, ColQwen2, ColQwen2.5이다. 추가로 재학습 없이 prefix만 잘라 평가하는 direct dimension truncation baseline을 비교 대상으로 삼는다.

PDF p. 5

지표

주요 지표는 nDCG@5(표에서 ×100 표기)이며 Recall@1, Recall@5를 함께 보고한다. budget 탄력성은 Retention(ℓ,d) = nDCG@5(ℓ,d) / nDCG@5(L,D) (%, Eq.8)로 측정한다. 효율성 지표로는 bf16 page-vector footprint 기준 indexing storage(GiB), indexing latency(ms/page, early exit 포함), query latency 및 Online latency(ms/query, Online = Query + 1000-candidate MaxSim scoring)를 사용한다. 논문에 따르면 이 효율성 측정은 PaliGemma 기준 단일 NVIDIA H100 80GB HBM3 GPU, Intel Xeon Platinum 8468 CPU, PyTorch 2.9.1+cu128, CUDA 12.8, Python 3.10.19 환경에서 bf16 precision으로 수행되었으며, 절대 latency 수치는 이 하드웨어·소프트웨어 조건에 종속된다.

PDF p. 5

미확인 사항

실제 학습에 사용된 learning rate, optimizer, epoch/step 수, batch size B, 최종 선택된 λ와 Tdist의 구체적 수치는 본문에 숫자로 명시되어 있지 않다. Figure 7과 Table 6은 선택된 설정 대비 상대적인 sweep 변형(T=0.1, T=0.3 등)만 비교한다. 확인되지 않음.

주요 결과와 의미

Middle-layer 저차원 retention (Table 1)

종합 정리

분석자 해석

설계의 핵심

MM-Matryoshka의 핵심은 새로운 인덱스 구조나 압축 알고리즘이 아니라, 기존 late-interaction 인터페이스를 그대로 둔 채 학습 목적함수만으로 하나의 모델을 구성된 2D budget grid 위에 펼치는 데 있다. 추론 인터페이스가 MaxSim으로 고정되어 있으므로 기존 ColPali류 서빙 스택에 비교적 낮은 마찰로 얹을 수 있다는 것이 구조적 이점이다(분석자 해석).

PDF p. 3, 4

두 손실의 분업

Ablation 결과를 종합하면 dimension-level MRL은 폭 축의 열화를 완화하고 layer-level distillation은 깊이 축의 붕괴를 막는, 서로 대체 불가능한 분업 관계에 있다. 어느 하나만으로는 2D 탄력성을 얻을 수 없다는 것이 실험적으로 뒷받침된다(분석자 해석, 근거: Figure 6b/6c).

PDF p. 7, 8

실무적 함의

동일 모델이 여러 비용 지점을 노출하므로, 배포 환경별로 별도 모델을 학습·관리할 필요가 줄어든다. 다만 layer 축 절감이 실제 latency 이득으로 이어지려면 early-exit 실행에 대한 시스템 레벨 지원이 필요하다는 점을 저자 스스로 한계로 명시하며, 보고된 latency/speedup 수치도 특정 GPU·소프트웨어 스택에서 측정된 값이라는 점을 감안해야 한다.

논문의 강점

  • 분석자 해석 두 개의 손실이 각각 width 축과 depth 축을 담당하도록 명확히 분리 설계되었고, ablation(Figure 6b/6c)에서 각 손실을 제거했을 때 담당 축에서만 큰 열화가 나타나 설계 의도가 실험적으로 검증된다.
    PDF p. 7, 8
  • 분석자 해석 학습과 추론이 동일한 MaxSim late-interaction 함수를 사용하므로, budget 변경이 검색 인덱스 구조나 스코어링 커널의 교체를 요구하지 않는다. 기존 ColPali류 서빙 스택에 낮은 마찰로 적용 가능한 구조이다.
    PDF p. 3, 4
  • 논문 주장 PaliGemma, Qwen2-VL, Qwen2.5-VL 세 backbone과 ViDoRe v1·v2의 14개 데이터셋에서 검증되어, 특정 아키텍처에 종속된 트릭이 아님을 뒷받침한다.
    PDF p. 2, 5
  • 분석자 해석 정확도(nDCG@5, Recall, Retention)와 효율성(저장소 GiB, indexing/query/online latency)을 함께 보고하고 Appendix에 측정 하드웨어·소프트웨어 환경 및 원자료 표(Table 4)를 명시하여 trade-off를 재현·검증하기 쉽다.
    PDF p. 5, 12, 13
  • 논문 주장 정량 결과를 정성적 사례 연구(Figure 5, Table 5)로 보완하여, 고압축 budget에서도 정답 페이지를 상위에 올리는 구체적 사례를 제시한다.
    PDF p. 7, 8, 12, 13

한계

  • 저자 명시 한계 Hardware-aware Deployment Optimization: 프레임워크는 이론적으로 depth-level elasticity를 지원하지만, 다양한 특수 하드웨어에서 최적의 end-to-end 추론 속도 향상을 얻으려면 early-exit 실행을 위한 맞춤형 시스템 레벨 엔지니어링이 필요하다. 저자들은 TensorRT나 vLLM 같은 inference engine에 통합되는 배포 구성을 향후 개발할 계획이라고 밝힌다.
    PDF p. 9
  • 저자 명시 한계 Dynamic Budget Allocation: 현재 방법은 추론 시 사용자가 정적으로 지정한 2D budget에 의존하며, 개별 질의나 페이지의 복잡도에 따라 연산량을 동적으로 조정하지 않는다. 저자들은 샘플별 최소 충분 layer-dimension 쌍을 예측하는 instance-aware routing mechanism을 향후 과제로 제시한다.
    PDF p. 9
  • 저자 명시 한계 Extension to Temporal Modalities: 실증 범위가 공간적(spatial) 문서 이미지에 한정되며, 연속적인 시간적 modality에 대한 적응력은 검증되지 않았다. 저자들은 video-language model로의 확장을 향후 연구로 제시한다.
    PDF p. 9
  • 분석자 해석 본문(7페이지)의 추론 속도 향상 수치(1.78×, 2.17×)와 Appendix B Table 4의 Online speedup(각각 1.61×, 2.12×)이 동일 budget에 대해 서로 다르며, 논문에 이 차이에 대한 설명은 확인되지 않는다. 두 수치 모두 단일 NVIDIA H100 GPU, bf16 환경에서 측정된 것이며, 보고된 속도 이득을 인용할 때 어느 정의를 쓰는지 주의가 필요하다.
    PDF p. 7, 12, 13
  • 분석자 해석 layer 축 절감이 저장소 절감으로 직결되지 않는다. dimension 축소는 index 크기를 직접 줄이지만, layer 축소의 이득은 encoding 연산량과 latency에 국한되며 그마저도 early-exit 구현과 특정 GPU/소프트웨어 환경(단일 H100, PyTorch 2.9.1+cu128, CUDA 12.8, bf16)에서의 측정을 전제로 한다(4페이지 서술 및 Table 4의 latency-only 개선에 근거한 분석자 추론).
    PDF p. 4, 12, 13
  • 분석자 해석 target layer set S와 prefix set D가 backbone별로 수작업 선택되며 선택 절차의 원칙이 본문에 정식화되어 있지 않아, 새로운 backbone에 적용할 때 grid 설계 비용이 발생할 수 있다(분석자 추론).
    PDF p. 4
  • 분석자 해석 학습 hyperparameter(learning rate, optimizer, epoch/step, batch size B, 최종 λ 및 Tdist 값)가 숫자로 명시되지 않아 완전한 재현이 어렵다. 확인되지 않음.

Visual Document Retrieval (VDR)

Multi-vector retriever / Late interaction
질의와 문서를 각각 다수의 token vector로 표현하고, 검색 시점에 vector 간 상호작용으로 점수를 계산하는 방식. ColPali류 VDR 리트리버의 기본 구조이다. PDF p. 1, 2
MaxSim
late interaction의 점수 함수로, query token vector마다 가장 유사한 page token vector와의 내적 최댓값을 구해 평균한다(Eq.4). 연산 비용은 query vector 수·page vector 수·차원 수에 비례한다. PDF p. 1, 3
Matryoshka Representation Learning (MRL)
임베딩의 앞부분 prefix만 잘라도 유효한 표현이 되도록 학습하는 기법. 전통적 MRL은 vector width만 제어한다. PDF p. 1
2D budget (ℓ,d)
encoder layer 깊이 ℓ과 prefix dimension d의 쌍으로 정의되는 비용 지점. MM-Matryoshka는 단일 모델이 backbone별로 구성된 target layer set S와 prefix dimension set D로 이루어진 2D budget grid 위에서 동작하도록 학습한다. PDF p. 3
Prefix Truncation (τd)
token vector x∈R^D를 앞 d차원으로 자르고 x1:d/‖x1:d‖2로 재정규화하여 budgeted representation을 만드는 연산(Eq.3). PDF p. 3
Dimension-Level MRL Loss (Lprefix)
final layer에서 여러 prefix 차원 각각의 in-batch score matrix에 가중치 αd를 적용한 cross-entropy retrieval loss(Eq.5). 좁은 벡터의 즉시 사용 가능성을 학습한다. PDF p. 3, 4
Layer-Level MRL Loss / Score Distillation (Ldistill)
final layer를 teacher(stop-gradient)로 두고 intermediate layer(student)의 in-batch score 분포를 온도 Tdist의 KL divergence로 맞추는 손실(Eq.6). PDF p. 4
Retention
Retention(ℓ,d) = nDCG@5(ℓ,d) / nDCG@5(L,D) (%, Eq.8). full budget 대비 축소 budget이 유지하는 정확도 비율로, budget 탄력성의 핵심 지표이다. PDF p. 5
Early exit
encoder를 최종 layer까지 실행하지 않고 선택된 중간 layer에서 종료하는 실행 방식. indexing latency를 줄이지만 실제 이득을 얻으려면 시스템 레벨 지원이 필요하다. PDF p. 4, 9, 12
Online latency
Query latency에 1000개 후보에 대한 MaxSim scoring 시간을 더한 질의당 지연시간(ms/query). PDF p. 5, 12
ViDoRe v1 / v2
VDR 평가 benchmark. v1은 10개, v2는 4개 sub-dataset으로 구성되며 논문은 총 14개 데이터셋에서 평가한다. PDF p. 5
