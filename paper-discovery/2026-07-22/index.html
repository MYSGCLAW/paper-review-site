GORIO는 graph ANNS를 위한 GPU 중심 로컬 I/O를 NVMe-oF 기반 원격 스토리지로 확장한 시스템으로, query 진행·page-miss 감지·pending query 상태·resume 결정을 모두 GPU에서 처리하고 CPU는 NVMe-oF transport/completion proxy 역할만 수행한다. SIFT1M DiskANN 스타일 워크로드에서 기존 최신 remote-I/O 경로 대비 1.31배, 직접 remote page-cache 경로 대비 4.89배 빠르다. 학회 게재 여부는 미확인된 기술 보고서 수준의 arXiv preprint다.

GPU가 storage I/O 제어를 직접 주도해 host bottleneck을 없애는 설계는 candidate embedding을 원격/분리된 storage에서 GPU로 직접 가져오는 문제(질문 11, 27)에 적용 가능한 아이디어다.
