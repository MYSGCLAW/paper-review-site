billion-scale on-SSD graph 기반 ANN 검색에서 CPU 연산이 병목이 되는 문제를 다루며, GPU 중심·CPU 보조 아키텍처(GustANN)를 제안한다. GPU 메모리 효율적 커널로 GPU-SSD 동시성을 높이고, CPU-보조 전송으로 PCIe 대역폭 병목을 완화하며, SSD 간 부하 분산을 위한 pivot search를 도입한다. 기존 시스템 대비 처리량 2.50배, 비용 효율 2.62배 향상을 보고한다.

VeloANN은 SSD 기반 graph ANN 탐색에서 access locality 부족으로 인한 storage stall과 read amplification 문제를 다룬다. affinity 기반 데이터 배치로 관련 vector를 같은 page에 배치하고, record-level buffer pool로 hot record를 유지하며, coroutine 기반 비동기 I/O와 beam-aware prefetch를 결합한다. 기존 disk 기반 ANN 대비 처리량 최대 5.8배, 지연시간 3.25배 개선을 보고한다.

variable-length embedding의 block-aligned layout, hot/cold placement, prefetch 효과(질문 4, 11, 15)와 정확히 맞닿아 있어 multi-vector용 storage layout 설계에 참고할 기법이 많다.

Starling은 data segment 단위로 disk-resident graph index의 data layout과 탐색 전략을 최적화한다. locality를 높인 reordered disk graph와 in-memory navigation graph를 결합해 탐색 경로를 줄이고, block search 전략으로 disk I/O 낭비를 최소화한다. 수천만 벡터 규모에서 0.9 이상 recall과 밀리초 이하 지연시간을 달성하며 기존 대비 최대 98% 낮은 query latency를 보고한다.
