A Voronoi Cell Formulation for Principled Token Pruning in Late-Interaction Retrieval Models

검증 완료

Yash Kankanampati, Yuxuan Zong, Nadi Tomeh, Benjamin Piwowarski, Joseph Le Roux

2026 · Proceedings of the 49th International ACM SIGIR Conference on Research and Development in Information Retrieval (SIGIR '26), Melbourne, VIC, Australia

이 논문은 ColBERT 계열 late-interaction retrieval 모델의 index storage 문제를 해결하기 위한 token pruning을 기하학(geometry) 문제로 재정의한다. 핵심 아이디어는 문서 토큰 벡터 d_i에 대해 'd_i가 최대 내적(max inner product)을 얻는 쿼리 벡터들의 집합'을 Voronoi cell V_i로 정의하는 것이다(Eq. 5, p.3). 어떤 토큰을 제거했을 때 발생하는 손실은, 그 토큰의 Voronoi cell 안에 있는 쿼리에 대해서만 MaxSim 점수가 떨어지므로, 해당 cell 내부에서 'first-best 매치와 second-best 매치의 차이'의 기댓값으로 정확히 표현된다(Eq. 6-7, p.3). 즉 pruning error가 임의의 휴리스틱이 아니라 닫힌 형태의 기대값으로 정식화되며, error가 정확히 0인 토큰만 제거하도록 제한하면 기존 lossless pruning 문제로 환원된다(p.3). 이 적분은 계산 불가능하므로 저자들은 unit sphere에서 균일하게 샘플링한 N개의 가상 쿼리 벡터로 Monte Carlo 추정을 수행하고(Eq. 8, p.4), 매 iteration마다 error가 가장 작은 토큰 하나를 제거한 뒤 남은 토큰의 error를 재계산하는 greedy iterative 알고리즘(Algorithm 1, p.5)을 사용한다. 이 연산은 대규모 행렬 곱과 argmax로만 구성되어 GPU에서 병렬화가 쉽다는 점이 실용적 이점이다. 결과적으로 MS MARCO dev set에서 토큰의 50%만 남기고도 MRR@10 38.9를 기록해 unpruned ColBERTv2-e2e(39.7)의 98.0%를 보존했고(Table 1, p.7), 같은 조건의 learning-free 방법(First-p 37.7, Att. Score 36.0, IDF 32.6)을 모두 앞섰다. 속도 면에서는 10,000개 문서 pruning에 12.0초(H200 GPU)가 걸려 LP-Pruning의 1,474.3초(CPU) 대비 극적으로 빠르며, 8.8M 전체 컬렉션 기준 약 3시간 대 약 15일로 추정된다(p.7). 또한 6%만 남기는 aggressive pruning에서 LP-Pruning의 nDCG@10이 0.46으로 붕괴하는 동안 제안 방법은 0.67을 유지했고(Figure 4, p.8), 추정된 mean error(ME)와 실제 nDCG@10 사이에 R²=0.99(TREC-DL 평균)의 선형 관계가 성립해 pruning 예산 결정을 사전에 안내할 수 있는 진단 도구로도 쓰일 수 있음을 보였다(Figure 7, p.9-10).

BiSCo-LLM: Lookup-Free Binary Spherical Coding for Extreme Low-Bit Large Language Model Compression

검증 완료

Yuantian Shao, Peisong Wang, Zhilei Liu, Chuangyi Li, Yuanteng Chen, Pengcheng Xie, Yiwu Yao, Zhihui Wei, Jian Cheng

2026 · arXiv preprint (arXiv:2607.08643v1, cs.LG) — 정식 게재 학회/저널은 확인되지 않음

BiSCo-LLM은 LLM weight를 weight당 2 bit 수준으로 압축하면서도 vector quantization(VQ)이 요구하는 explicit codebook과 index lookup을 완전히 제거하는 것을 목표로 하는 storage-aware 압축 파이프라인이다. 핵심 아이디어는 binary spherical quantization(BSQ)이다. d차원 weight chunk를 encoder로 latent vector에 사상하고 unit hypersphere로 정규화한 뒤 sign 함수로 이진화하면, centroid 테이블을 저장하지 않고도 각 chunk가 하나의 binary code(sign pattern)로 표현되며, 복원은 저장된 codebook 조회가 아니라 모든 chunk가 공유하는 compact neural decoder의 forward 연산으로 수행된다(lookup-free). 저자들은 여기에 세 가지 장치를 추가한다. 첫째, 단일 스테이지 code 길이를 늘리는 것이 비효율적임을 실측(Qwen3-8B qproj에서 b=64일 때 레이어당 실제 점유 code pattern 중앙값이 524,286개로 2^64 공간의 약 2.84×10^-14에 불과)으로 보이고, 대신 base 복원이 남긴 residual에 두 번째 BSQ codec을 할당하는 second-stage residual coding을 도입한다. 둘째, activation 통계와 weight 크기를 결합한 sensitivity score로 선정한 소수(ρ=1%) 채널만 8-bit로 따로 저장하는 protected-channel path를 둔다. 셋째, layer 단위 local loss 감소가 조립된 모델의 성능 회복을 보장하지 않는다는 진단(local loss는 36개 block 모두에서 평균 27.26% 감소했지만 prefix 평가 downstream 정확도는 70.34%에서 최종 53.36%로 붕괴)을 근거로, module category(q/k/v/o/gate/up/down_proj) 단위로 교체와 recovery distillation을 번갈아 수행하는 category-wise 스케줄을 채택한다. 압축 크기는 code stream, decoder parameter, 8-bit payload, LoRA adapter, metadata를 모두 더한 real storage budget(Stot)으로 명시적으로 계산된다. Qwen3-8B에서 WikiText-2 perplexity 10.18(FP16/BF16 9.73), 보고된 7개 task 평균 정확도 68.05(FP16/BF16 69.92, 격차 1.87점)로 AQLM·QuIP#·UniSVQ 등 2-bit VQ baseline을 상회한다고 보고한다. 다만 Qwen3-8B의 HellaSwag 결과와 LLaMA3-8B 전체 결과가 아직 비어 있어 논문은 진행 중인 초안 성격을 갖는다.

Col-Bandit: Query-Time Top-K Estimation for Late-Interaction Retrieval

검증 완료

Roi Pony, Adi Raz Goldfarb, Oshri Naparstek, Idan Friedman, Udi Barzelay, Eli Schwartz

2026 · arXiv preprint (arXiv:2602.02827v2 [cs.IR])

Col-Bandit은 ColBERT류 late-interaction retriever의 rerank 단계를 가속하는 query-time 알고리즘이다. 핵심 관찰은 N개의 candidate document와 T개의 query token이 만드는 implicit N×T MaxSim Matrix H에서, 각 문서의 최종 점수 Si가 해당 행의 row-sum이라는 점이다(Eq. 3-5, p.3-4). 기존 exhaustive scorer는 이 행렬의 모든 cell을 계산하지만, 논문은 top-K 문서 집합을 '식별(identify)'하는 데에는 행렬 전체가 필요하지 않다고 본다(Figure 2, p.4). Col-Bandit은 이를 finite-population Top-K identification 문제로 정식화하고, 부분적으로 공개된(partially revealed) cell로부터 문서별 신뢰구간 [LCBi, UCBi]를 유지한다. 이 구간은 support [a,b]에서 나오는 deterministic hard bound와 empirical Bernstein–Serfling inequality(Bardenet and Maillard, 2015) 기반의 variance-adaptive radius r_i^eff(Eq. 7)를 결합한 hybrid interval이다(p.4). 매 round마다 공유된 random permutation π를 따라 B(=4)개의 query token cell을 active set 전체에 대해 한 번의 SIMD pass로 공개하고, UCB가 K번째로 큰 LCB(threshold τr) 아래로 떨어진 문서를 영구 제거한다(Algorithm 1, p.5). 남은 K+M명의 survivor는 Full-MaxSim과 동일한 fused kernel로 전체 T개 token에 대해 정확히 rescore되므로, 최종 점수는 exhaustive baseline과 bit-identical하다(p.5). relaxation knob αef는 compute–fidelity trade-off를 조절하는 단일 파라미터로, αef=1에서는 δ-PAC guarantee(Theorem 1)를 제공하고 배포 기본값 αef=0.2는 formal certificate 없이 calibrated된 운용점을 제공한다(p.5, p.9). 실험적으로 index 수정이나 retraining 없이 BEIR 5개 corpus와 REAL-MM-RAG 4개 corpus(최대 2.68M documents)에서 모든 corpus Overlap@5 ≥ 0.90을 유지하면서 서버 CPU(AMD EPYC 7763, AVX2)에서 single-thread 평균 7.0×, Apple M1 Max(NEON)에서 12.9× wall-clock speedup을 보고한다(p.2, p.8).

ColBERTSaR: Sparsified ColBERT Index via Product Quantization

검증 완료

Eugene Yang, Andrew Yates, Dawn Lawrie, James Mayfield, Saron Samuel, Rohan Jha

2026 · SIGIR '26 (49th International ACM SIGIR Conference), Melbourne, VIC, Australia; arXiv:2606.05568v1 [cs.IR]

ColBERTSaR는 ColBERT 계열 multi-vector dense retrieval의 핵심 확장성(scalability) 문제인 거대한 index 크기를, PLAID가 저장하던 residual 벡터를 완전히 제거하는 residual-free product quantization으로 해결한다. 저자들은 MaxSim 점수를 anchor(centroid)와 query token의 내적만으로 근사하는 Score_S(Equation 3)를 유도하고, 이 식이 anchor를 sparse vocabulary로 보는 dynamic TF-IDF 형태임을 보여 embedding quantization을 적용한 ColBERT가 scoring mechanism을 제외하면 learned sparse retrieval(LSR)과 이론적으로 동등함을 확립한다. 구현은 retrieval 모델을 재학습하지 않는 PLAID의 drop-in replacement로서, K-means로 학습한 anchor에 각 문서 토큰을 할당해 inverted index와 forward index를 만들고 nprobe 기반 2단계 검색을 수행한다. proof-of-concept 실험에서 ColBERTSaR index는 1-bit PLAID 대비 50~70% 작으면서 monolingual(BEIR), CLIR·MLIR(NeuCLIRBench, NeuCLIRTech)에서 경쟁적인 retrieval 효과성을 유지한다.

FLASH-MaxSim: IO-Aware Fused Kernels for Late-Interaction Retrieval

검증 완료

Roi Pony, Daniel Ezer, Adi Raz Goldfarb, Idan Friedman, Oshri Naparstek, Udi Barzelay

2026

이 논문은 ColBERT/ColPali로 대표되는 late-interaction retrieval의 핵심 연산인 MaxSim(쿼리 토큰마다 문서 토큰에 대해 max를 취하고, 다시 쿼리 토큰에 대해 sum하는 연산)을 GPU에서 효율적으로 계산하는 IO-aware fused 커널 FLASH-MaxSim(FM)을 제안한다. 표준 PyTorch 구현은 [Nq, B, Lq, Ld] 크기의 query×document 유사도 텐서 S를 HBM에 전부 만든 뒤 max·sum으로 줄이고 버리는데, 이 텐서는 오직 reduce되기 위해서만 존재하면서 추론 시 후보 집합 크기와 대조학습(contrastive training) 시 배치 크기를 제한하는 메모리 병목이 된다(예: ColPali 10K 문서에서 FP16 기준 21GB). 저자들은 FlashAttention의 IO-aware tiling 원리를 MaxSim이라는 hard-selection reduction(한 축에 max/min/arg, 다른 축에 sum)에 적용하여, forward에서는 유사도 텐서를 전혀 만들지 않고 SRAM 안에서 tile 단위 running max로 점수만 계산하고, backward에서는 forward가 저장한 argmax를 sparse bipartite routing graph로 취급하는 inverse-grid CSR 구조로 gradient 텐서 없이 정확한 gradient를 복원한다. 그 결과 추론 peak memory를 최대 9× 낮추고 학습 peak memory를 두 자릿수 배(orders of magnitude) 줄이면서, 모든 shape에서 naive PyTorch보다 빠르고(최대 A100 3.8×/H100 4.7×) FP32 정밀도 프로토콜 하에서 순위(ranking)를 BEIR·REAL-MM-RAG에서 nDCG@10 소수점 4자리까지 보존한다. 여기에 저장공간을 절반으로 줄이는 approximate INT8 경로와 d>512에서의 SRAM-spill을 없애는 split-d forward가 서빙 지향 확장으로 추가된다.

GEM: A Native Graph-based Index for Multi-Vector Retrieval

검증 완료

Yao Tian, Zhoujin Tian, Xi Zhao, Ruiyuan Zhang, Xiaofang Zhou

2026

GEM은 multi-vector retrieval(질의와 문서를 모두 토큰 단위 벡터들의 집합, vector set으로 표현하는 검색)을 위해 vector set 자체를 그래프의 정점으로 삼는 최초의 native graph index이다. 기존 방식은 (1) 모든 개별 벡터를 단일 ANN 인덱스에 pooling하거나(PLAID, IGP 계열), (2) vector set을 단일 벡터로 사상하거나(MUVERA의 FDE), (3) LSH sketch로 압축(DESSERT)하는 식으로 단일 벡터용 인덱스를 재사용했는데, 이들은 set-level semantics를 보존하지 못하거나 pruning이 약해 느리다. GEM의 핵심 통찰은 그래프 기반 검색이 전제하는 metric 성질(삼각부등식)을 MaxSim/Chamfer(CH) 유사도가 만족하지 않는다는 점이며, 이를 metric decoupling으로 해결한다. 즉 그래프 구축(edge weight)에는 삼각부등식을 만족하고 CH(Q,P) <= EMD(Q,P)를 만족하는 Earth Mover's Distance(EMD)를 쓰고, 실제 relevance 산출과 탐색에는 원래의 Chamfer distance를 쓴다. 여기에 TF-IDF 기반 set-level clustering으로 각 set을 소수의 유의미한 cluster에만 할당해 index redundancy를 줄이고(예: MSMARCO에서 set당 평균 cluster membership 43.8 -> 2.9), cluster 내부 local proximity graph를 cluster 간 bridge로 이어 붙여 globally navigable한 dual graph를 만든다. Chamfer 상으로는 가깝지만 EMD 그래프 상 멀리 떨어진 쌍은 학습 pair를 재사용한 semantic shortcut으로 직접 연결한다. 인덱싱과 검색 모두 centroid codebook 기반의 quantized EMD(qEMD)/quantized Chamfer(qCH) lookup으로 set-to-set 거리 계산 비용을 낮추며, 질의 시에는 cluster filtering -> multi-entry 병렬 beam search -> cluster 기반 early pruning -> exact Chamfer rerank 순으로 처리한다. MSMARCO, LoTTE, OK-VQA, EVQA 네 벤치마크에서 GEM은 모든 품질 지표(R@100, S@100, MRR@10)에서 최고 성능을 내면서 지연시간도 가장 낮았고(MSMARCO 140ms vs PLAID 352ms), 동일 MRR@10 기준 최대 16배 speedup과 함께 index size는 PLAID/IGP와 비슷하거나 더 작다.

MUVERA는 이 MV 유사도 검색을 단일 벡터(single-vector, SV) 최대 내적 검색(Maximum Inner Product Search, MIPS)으로 환원(reduction)하는 검색 기법이다. 핵심은 질의와 문서 각각의 벡터 집합을 데이터 비의존적(data-oblivious) 랜덤 변환으로 하나의 고정 차원 벡터로 인코딩하는 Fixed Dimensional Encoding(FDE)이다. 두 FDE의 내적이 원래의 Chamfer 유사도를 근사하므로(Theorem 2.1의 epsilon-additive 근사), 문서 FDE를 기존의 고도로 최적화된 MIPS 인덱스(DiskANN)에 색인하고, 질의 FDE로 상위 후보를 검색한 뒤 원래 Chamfer 유사도로 한 번만 재순위(re-rank)하면 된다. 저자들은 FDE가 기존 SV heuristic과 동일한 recall을 2-5배 적은 후보만으로 달성하고, BEIR 데이터셋 전반에서 PLAID 대비 평균 recall 10% 향상과 평균 latency 90% 감소를 얻는다고 보고한다.

ESPN: Memory-Efficient Multi-Vector Information Retrieval

검증 완료

Susav Shrestha, Narasimha Reddy, Zongwang Li

2023 · Under review (arXiv:2312.05417v1)

ColBERT류의 멀티벡터(late interaction) 검색 모델은 문서를 토큰 단위 멀티벡터로 인코딩하고 MaxSim 연산으로 유사도를 계산해 단일 벡터·lexical 모델보다 검색 품질(MRR@10)을 크게 높이지만, 재랭킹용 임베딩 인덱스 크기가 BM25 대비 한 자릿수 이상(예: ColBERTv1은 210배, ColBERTv2·ColBERTer도 26-34배) 커지면서 대규모 데이터셋에서 메모리 확장성이 심각한 병목이 된다.\n이 논문은 ESPN(Embedding from Storage Pipelined Network)을 제안한다. 핵심은 방대한 bag-of-word(BOW) 재랭킹 임베딩 테이블 전체를 SSD로 오프로드하되, Nvidia GPUDirect Storage(GDS)로 SSD에서 GPU 메모리로 직접(peer-to-peer) 데이터를 전송하고, ANN(근사 최근접 이웃) 탐색의 특성을 이용한 소프트웨어 프리페처(prefetcher)로 스토리지 지연을 ANN 연산과 오버랩시키는 것이다. 프리페처는 90%를 넘는 히트율을 달성하고, 조기 재랭킹(early re-ranking)과 부분 재랭킹(partial re-ranking)을 결합해 대역폭 요구량을 줄인다.\n결과적으로 ESPN은 전체 리트리벌 인덱스의 6-19%만 메모리에 유지하면서도 완전 메모리 기반 솔루션 대비 1.02배 수준의 end-to-end 지연시간을 유지하고, 메모리 요구량을 quantization factor에 따라 5-16배 줄이며, 메모리 용량 근처에서 mmap 대비 3.1-3.9배 빠른 지연을 달성한다.
