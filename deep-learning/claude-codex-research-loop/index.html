Claude-Codex 딥러닝 자료 확장 루프

목표

이 루프는 Claude가 수집한 런타임 자료를 Codex가 단순 검증하는 흐름이 아니라, 두 에이전트가 서로의 산출물을 계속 확장하는 방식으로 자료를 키우기 위한 구조다. 매 턴은 반드시 새 정보, 더 깊은 구체화, 관련 맥락, 오류 수정 후 보강, 다음 조사 질문 중 하나 이상을 남겨야 한다.

핵심 종료 조건은 내용상 완료가 아니라 실행상 중단이다. 둘 중 하나가 오류를 내거나, 토큰이 부족하거나, 외부 명령이 실패하거나, 명시적인 중단 신호가 생길 때까지 자료 생성은 계속된다.

반복 구조

초기 주제 / 조사 목표
        ↓
Claude: 1차 자료 수집 및 정리
        ↓
Codex: 추가 조사 / 구체화 / 비교 / 누락 영역 확장
        ↓
Claude: 검토와 동시에 새 사실, 반례, 출처 후보 보강
        ↓
Codex: 다시 하위 주제 조사, 구조화, 다음 질문 생성
        ↓
오류, 토큰 부족, 실행 실패, 중단 신호까지 반복

에이전트 규칙

에이전트금지되는 행동반드시 해야 할 행동
Claude검증 완료, 추가 없음, 변함 없음으로 끝내기새 사실 추가, 특정 항목 구체화, 관련 맥락 보강, 다음 확장 지점 제시
Codex맞다/틀리다 판정만 하고 종료하기오류가 있으면 수정한 뒤 관련 내용을 더 추가하고, 비교표나 조사 질문으로 다음 턴을 열기

강제 확장 조건

각 턴은 아래 항목 중 최소 2개 이상을 포함해야 한다.

  1. 새로 추가한 사실
  2. 더 구체화한 부분
  3. 관련 배경 또는 시스템 맥락
  4. 잠재적 오류와 수정 내용
  5. 출처 확인용 검색 키워드
  6. 다음 에이전트가 이어서 확장할 질문

아래 문구가 나오면 해당 턴은 실패로 간주하고 다시 요청한다.

1Codex루프 설계검증 루프가 아니라 증식 루프로 설계하고, 금지 문구 필터와 출력 포맷, 반복 제어 로직을 정의했다.
2사용자UI 요구 추가홈페이지에 딥러닝 탭을 만들고, 변화와 응답 누적을 볼 수 있도록 구성해 달라고 요청했다.
3Codex사이트 반영딥러닝 컬렉션을 홈페이지 탭에 추가하고, 이 문서를 첫 번째 누적 로그로 등록했다.

다음 루프에서 확장할 지점

  1. 실제 Claude 실행 로그를 저장할 파일 형식: jsonl, markdown append, turns.json 중 어느 쪽이 좋은지 비교한다.
  2. Codex가 생성한 추가 조사 결과를 Claude 입력으로 전달할 때 압축 요약과 원문 로그를 어떻게 나눌지 설계한다.
  3. 홈페이지 카드에서 턴 수, 마지막 작성자, 마지막 변화 유형을 바로 보이게 하는 메타데이터 확장을 추가한다.
  4. 런타임 자료 수집 작업에 이 루프를 적용해 CUDA, Level Zero, PJRT, UPMEM 항목별 누적 로그를 분리한다.
